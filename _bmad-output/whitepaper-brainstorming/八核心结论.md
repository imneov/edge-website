# 八、核心结论

## 叙事主线

白皮书的核心论点不是"我们有什么技术"，而是**"应用在边缘的每个阶段面对什么问题，平台如何解决"**。

**价值重心从硬件前置投入转向应用持续释放。** 这是整篇白皮书的立论基础。

围绕应用的基础设施非常广泛，包括硬件接入、算力分配、团队协作、应用打包分发、边缘部署、离线运行、设备交互、监控告警等。每一个技术能力都不是孤立存在的，而是应用在某个生命周期阶段必须面对的具体场景的解法。

## 章节结构：应用生命周期主线

白皮书以应用的完整旅程为主线，分为四个部分：

| 部分 | 章节 | 核心问题 |
|------|------|---------|
| **第一部分：为什么** | Ch1 执行摘要 + Ch2 范式转移 | 为什么价值在应用层？ |
| **第二部分：应用生命周期** | Ch3-Ch9（7 章） | 应用从诞生到运行的每个阶段怎么解决？ |
| **第三部分：架构纵深** | Ch10 平台架构 | 这些能力是怎么统一在一个架构下的？ |
| **第四部分：论证与落地** | Ch11-Ch13 + 附录 | 跟竞品比怎样？行业怎么用？怎么部署？ |

## 七个生命周期阶段

| 阶段 | 章节 | 应用面对的问题 | 平台的解法 |
|------|------|--------------|-----------|
| 基础设施就绪 | Ch3 | 硬件怎么快速接入？集群怎么建？镜像怎么到边缘？ | OTA 批量接入 + 集群联邦 + 分层镜像分发 |
| 算力就绪 | Ch4 | 异构 GPU 怎么统一使用？怎么切分共享？ | 四层 CRD 抽象 + MIG/HAMi 双模虚拟化 |
| 组织与安全 | Ch5 | 多团队怎么安全共享平台？ | 5 层权限模型 + 多租户隔离 |
| 应用供应链 | Ch6 | 应用怎么打包？怎么审核上架？ | Provisioner 插件 + 应用商店审核 |
| 应用部署 | Ch7 | 怎么部署到多个边缘？版本怎么批量切换？ | Topology-Based 部署 + 状态聚合 |
| 应用运行 | Ch8 | 断网了怎么办？怎么操作 IoT 设备？ | 离线自治 + IoT CRD 云原生化 |
| 应用可观测 | Ch9 | 怎么知道应用健康不健康？ | 291+ 指标 + 多租户隔离 + 告警 |

## 关键纠正

在本次结构调整中，纠正了两个重要的定位错误：

1. **OTA 的正确定位**：OTA 解决的是**硬件快速接入**的问题（远程装驱动、配环境、批量纳管），不是应用更新。应用的批量更新通过 ApplicationDeployment 的版本切换实现。
2. **IoT 的正确定位**：IoT 设备管理是应用**运行时与物理设备交互**的能力，不是独立的管理系统。它属于应用运行阶段。

## 与竞品的本质区别

**竞品讲的故事**：我们支持什么硬件、管理多少节点、提供什么运行时。
**我们讲的故事**：你的应用在边缘的每个阶段都有解法，无论底层硬件怎么变。

这不是措辞差异，而是产品哲学的根本不同——它决定了白皮书的组织方式不是按基础设施分类（算力一章、网络一章），而是按应用面对的问题展开。
