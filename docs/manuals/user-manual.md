## 1. 快速入门

### 1.1 系统登录

#### 操作路径

浏览器访问平台控制台地址（默认 `https://<平台域名>`），进入登录页面。

#### 步骤说明

1. 在浏览器地址栏输入平台管理控制台的访问地址。
2. 在登录页面输入 **用户名** 和 **密码**。
   - 首次登录使用管理员分配的初始账号和密码。
3. 点击 **登录** 按钮。
4. 若为首次登录，系统会强制要求修改初始密码：
   - 输入当前密码（初始密码）。
   - 输入新密码并确认（需满足密码策略要求：至少 8 位，包含大小写字母、数字和特殊字符）。
   - 点击 **确认修改**。
5. 修改密码成功后，系统自动跳转至控制台首页。

#### 注意事项

- 连续输入错误密码超过 5 次，账号将被锁定 30 分钟，请联系管理员解锁或等待自动解锁。
- 建议使用 Chrome、Firefox 或 Edge 浏览器，版本不低于最近两个主要版本。
- 登录后如长时间未操作，会话将自动过期，需要重新登录。

---

### 1.2 控制台概览

#### 操作路径

登录后自动进入 **控制台首页（Dashboard）**。

#### 页面说明

控制台首页提供平台核心指标的全局视图，包含以下区域：

| 区域 | 说明 |
|------|------|
| **顶部导航栏** | 包含平台 Logo、全局搜索、消息通知、用户头像（个人设置/退出登录） |
| **左侧菜单栏** | 按功能模块组织的导航菜单，包括集群管理、应用管理、监控、OTA 等 |
| **工作空间切换** | 顶部可切换当前所在的工作空间，不同工作空间下资源相互隔离 |
| **资源概览卡片** | 显示当前工作空间下的集群数量、节点总数、运行中应用数、告警数等 |
| **快捷操作** | 常用功能的快速入口：创建集群、部署应用、查看告警等 |

#### 个人设置

点击右上角用户头像，可进行以下操作：

- **个人信息**：查看和修改用户名、邮箱、手机号等基本信息。
- **修改密码**：修改当前登录密码。
- **偏好设置**：配置语言（中文/English）、时区、主题等。
- **退出登录**：安全退出当前会话。

#### 注意事项

- 首页数据根据当前用户的权限范围展示，不同角色看到的内容可能不同。
- 工作空间切换后，左侧菜单和主内容区将刷新为对应工作空间的资源视图。

## 2. 集群管理

### 2.1 查看集群

#### 操作路径

左侧菜单 → **集群管理** → **集群列表**

#### 步骤说明

1. 点击左侧菜单中的 **集群管理**，展开子菜单。
2. 点击 **集群列表**，进入集群列表页面。
3. 页面以列表形式展示当前用户有权限查看的所有集群，包含以下信息：

| 字段 | 说明 |
|------|------|
| 集群名称 | 集群的唯一标识名称 |
| 集群状态 | 运行中 / 异常 / 离线 / 初始化中 |
| 集群类型 | Host 集群 / 边缘集群 |
| 边缘框架 | KubeEdge / OpenYurt / vCluster |
| 节点数量 | 集群中的节点总数 |
| Kubernetes 版本 | 集群运行的 K8s 版本 |
| 创建时间 | 集群注册到平台的时间 |

4. 支持按集群名称搜索和按状态筛选。
5. 点击集群名称可进入 **集群详情** 页面，查看集群的详细信息、节点列表、资源使用情况等。

#### 集群详情页

集群详情页包含以下 Tab 页签：

- **概览**：集群基本信息、资源使用率（CPU / 内存 / 存储）、节点状态统计。
- **节点列表**：集群中所有节点的列表及状态。
- **节点组**：节点的逻辑分组管理。
- **项目**：集群下的项目（Namespace）列表。
- **集群设置**：集群的高级配置选项。

#### 注意事项

- 集群列表只显示当前用户有权限访问的集群。
- 集群状态为"异常"时，建议检查集群节点的网络连通性和组件运行状态。

---

### 2.2 创建集群

#### 操作路径

左侧菜单 → **集群管理** → **集群列表** → 点击 **创建集群**

#### 步骤说明

1. 在集群列表页面，点击右上角 **创建集群** 按钮。
2. 填写集群基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 集群名称 | 是 | 2-63 个字符，支持小写字母、数字和连字符，以字母开头 |
| 集群描述 | 否 | 集群的用途描述 |
| 边缘框架 | 是 | 选择 KubeEdge / OpenYurt / vCluster |
| Kubernetes 版本 | 是 | 根据所选框架自动列出支持的版本 |

3. 配置网络参数：
   - **Pod CIDR**：Pod 网络地址段（默认 `10.244.0.0/16`）。
   - **Service CIDR**：Service 网络地址段（默认 `10.96.0.0/16`）。
   - **CNI 插件**：选择容器网络插件（如 Flannel、Calico）。

4. 点击 **确认创建**。
5. 系统开始初始化集群，页面显示创建进度。
6. 集群创建完成后，状态变为 **运行中**。

#### 添加边缘节点

集群创建成功后，需要将边缘节点加入集群：

1. 进入集群详情页 → **节点列表** Tab。
2. 点击 **添加节点**。
3. 系统生成节点接入命令（包含 Token 和 API Server 地址）。
4. 在边缘节点服务器上执行该命令，完成节点注册。
5. 等待节点状态变为 **就绪（Ready）**。

#### 注意事项

- 创建集群前，请确认网络规划，Pod CIDR 和 Service CIDR 不能与已有网络地址段冲突。
- 边缘节点需要能够通过网络访问到管理集群的 API Server。
- 节点接入命令中的 Token 有时效限制（默认 24 小时），过期后需重新生成。
- 建议在生产环境中使用高可用集群配置（至少 3 个 Master 节点）。

### 2.3 节点组管理

#### 操作路径

左侧菜单 → **集群管理** → 选择目标集群 → **节点组** Tab

#### 概述

节点组（NodeGroup）是将集群中的节点按逻辑进行分组的机制，用于实现工作负载的拓扑调度。例如，可以将同一地理位置或同一业务用途的边缘节点划分为一个节点组。

#### 创建节点组

1. 在集群详情的 **节点组** Tab 页面，点击 **创建节点组**。
2. 填写节点组信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 节点组名称 | 是 | 节点组的唯一名称 |
| 描述 | 否 | 节点组的用途说明 |
| 节点选择 | 是 | 通过标签选择器或手动勾选将节点加入该组 |

3. 点击 **确认** 完成创建。

#### 管理节点组

- **查看节点组详情**：点击节点组名称，查看组内节点列表和资源使用情况。
- **编辑节点组**：点击操作列的 **编辑** 按钮，修改节点组名称、描述和节点成员。
- **删除节点组**：点击操作列的 **删除** 按钮，确认后删除节点组（不会删除组内节点）。

#### 注意事项

- 一个节点可以属于多个节点组。
- 节点组是应用拓扑分发的基础单元，部署应用时可以指定分发到特定的节点组。
- 删除节点组不会影响组内节点的运行状态和已部署的工作负载。
- 建议按照地理位置、业务场景或硬件类型进行节点分组。

## 3. 应用部署

### 3.1 镜像部署

#### 操作路径

左侧菜单 → **应用管理** → **应用商店** → 选择应用 → **部署**

或：左侧菜单 → **工作台** → **应用工作负载** → **创建工作负载**

#### 步骤说明（通过镜像创建工作负载）

1. 进入工作台的 **应用工作负载** 页面。
2. 点击 **创建工作负载** 按钮。
3. 选择工作负载类型：
   - **Deployment**：无状态应用，支持滚动更新和回滚。
   - **StatefulSet**：有状态应用，保证 Pod 的有序部署和持久化存储。
4. 填写基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 工作负载名称 | 是 | 工作负载的唯一标识 |
| 所属项目 | 是 | 选择工作负载所在的项目（Namespace） |
| 副本数 | 是 | Pod 副本数量（默认 1） |

5. 配置容器：
   - **镜像地址**：输入容器镜像的完整地址（如 `registry.example.com/app:v1.0`）。
   - **镜像拉取策略**：Always / IfNotPresent / Never。
   - **资源限制**：设置 CPU 和内存的 Request 和 Limit。
   - **端口映射**：配置容器端口（协议、容器端口号）。
   - **环境变量**：添加容器环境变量（支持直接输入、ConfigMap 引用和 Secret 引用）。
   - **存储挂载**：挂载持久卷或 ConfigMap/Secret 为文件。

6. 配置拓扑分发（可选）：
   - 选择目标集群和节点组。
   - 配置每个节点组的副本数和差异化配置。

7. 点击 **创建** 完成部署。

#### 注意事项

- 确保镜像地址可访问，如使用私有镜像仓库需提前配置镜像拉取凭证（ImagePullSecret）。
- 资源 Request 和 Limit 建议合理设置，避免资源不足导致 Pod 调度失败。
- 边缘场景下建议镜像拉取策略设为 `IfNotPresent`，减少重复拉取带来的带宽消耗。

---

### 3.2 Helm 部署

#### 操作路径

左侧菜单 → **应用管理** → **应用商店** → 选择 Helm 应用 → **部署**

#### 步骤说明

1. 进入 **应用管理** → **应用商店**。
2. 浏览或搜索目标应用。应用商店中的应用以 Helm Chart 形式打包。
3. 点击目标应用卡片，进入应用详情页。
4. 查看应用信息：
   - **应用描述**：功能说明和适用场景。
   - **版本列表**：可用的 Chart 版本。
   - **配置参数**：Chart 的 values 参数说明。
5. 点击 **部署** 按钮。
6. 填写部署配置：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 实例名称 | 是 | 应用实例的名称 |
| 所属项目 | 是 | 选择部署的目标项目 |
| Chart 版本 | 是 | 选择要部署的 Chart 版本 |
| 目标集群 | 是 | 选择部署的目标集群 |

7. 编辑配置参数：
   - 支持 **表单模式** 和 **YAML 编辑模式** 两种方式。
   - 表单模式：通过表单字段填写常用配置项。
   - YAML 模式：直接编辑 values.yaml 文件内容。

8. 点击 **确认部署**。
9. 系统执行 Helm Install，页面显示部署进度和结果。

#### 注意事项

- 部署前建议仔细阅读 Chart 的参数说明文档。
- 生产环境部署建议锁定具体的 Chart 版本，避免使用 `latest`。
- 如需自定义 Helm 仓库，可在 **镜像管理** → **镜像源管理** 中添加 Helm Chart 仓库。
- Helm 部署会自动管理关联的 Kubernetes 资源（Deployment、Service、ConfigMap 等）。

### 3.3 应用生命周期管理

#### 操作路径

左侧菜单 → **工作台** → **应用工作负载** → 选择目标工作负载

或：左侧菜单 → **应用管理** → **应用实例** → 选择目标实例

#### 应用状态说明

| 状态 | 说明 |
|------|------|
| 运行中 | 应用所有 Pod 处于正常运行状态 |
| 部署中 | 应用正在创建或更新中 |
| 异常 | 部分或全部 Pod 处于异常状态 |
| 已停止 | 应用副本数已缩为 0 |

#### 常用操作

**更新应用**

1. 在工作负载详情页，点击 **编辑** 按钮。
2. 修改镜像版本、环境变量、资源配置等。
3. 点击 **确认更新**，系统执行滚动更新。

**扩缩容**

1. 在工作负载详情页，点击 **扩缩容** 按钮。
2. 修改副本数量。
3. 点击 **确认**，系统自动增减 Pod 数量。

**回滚**

1. 在工作负载详情页，切换到 **版本记录** Tab。
2. 选择要回滚到的历史版本。
3. 点击 **回滚** 按钮并确认。

**重启**

1. 在工作负载详情页，点击 **更多操作** → **重启**。
2. 系统执行滚动重启（不会造成服务中断）。

**删除应用**

1. 在工作负载列表页，点击目标工作负载操作列的 **删除**。
2. 在确认弹窗中输入工作负载名称以确认删除。
3. 点击 **确认删除**。

#### 查看应用日志

1. 进入工作负载详情页 → **Pod 列表**。
2. 点击目标 Pod 的 **日志** 图标。
3. 选择容器（如有多个容器）。
4. 查看实时日志输出，支持关键字搜索和时间范围筛选。

#### 查看应用事件

1. 进入工作负载详情页 → **事件** Tab。
2. 查看工作负载相关的 Kubernetes 事件，包括调度、拉取镜像、容器启停等事件。

#### 注意事项

- 滚动更新期间，旧版本 Pod 和新版本 Pod 会共存，确保应用支持多版本并行运行。
- 删除应用是不可逆操作，请确认已备份重要数据。
- 扩缩容到 0 副本相当于停止应用，不会删除工作负载配置。
- 回滚操作仅回滚工作负载配置，不会回滚关联的 ConfigMap 和 Secret。

## 4. 配置管理

### 4.1 ConfigMap 管理

#### 操作路径

左侧菜单 → **工作台** → **配置管理** → **ConfigMap**

#### 概述

ConfigMap 用于存储非敏感的配置数据，如应用配置文件、环境变量、命令行参数等。ConfigMap 中的数据以键值对形式存储，可以被 Pod 以环境变量或挂载文件的方式引用。

#### 创建 ConfigMap

1. 在 ConfigMap 列表页面，点击 **创建 ConfigMap**。
2. 填写基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 名称 | 是 | ConfigMap 名称，同一项目下唯一 |
| 所属项目 | 是 | 选择 ConfigMap 所在的项目 |
| 描述 | 否 | 配置项的用途说明 |

3. 添加数据条目：
   - 点击 **添加数据**。
   - 输入 **键（Key）** 和 **值（Value）**。
   - 支持直接输入或上传文件作为值。
   - 可添加多个数据条目。

4. 点击 **创建** 完成。

#### 编辑 ConfigMap

1. 在 ConfigMap 列表中，点击目标 ConfigMap 名称进入详情页。
2. 点击 **编辑** 按钮。
3. 修改数据条目（添加、修改或删除键值对）。
4. 点击 **保存**。

#### 在工作负载中引用 ConfigMap

**方式一：作为环境变量**

在创建或编辑工作负载时，容器配置的环境变量部分：

1. 选择环境变量来源为 **ConfigMap**。
2. 选择目标 ConfigMap。
3. 选择要引用的 Key。
4. 设置环境变量名称。

**方式二：作为挂载文件**

在创建或编辑工作负载时，存储挂载部分：

1. 选择挂载类型为 **ConfigMap**。
2. 选择目标 ConfigMap。
3. 设置容器内挂载路径。
4. 可选择挂载全部 Key 或指定 Key。

#### 注意事项

- ConfigMap 更新后，以环境变量方式引用的 Pod 需要重启才能生效。
- 以挂载文件方式引用的 Pod，ConfigMap 更新后会自动同步（可能有短暂延迟）。
- ConfigMap 不适合存储敏感数据（如密码、密钥），敏感数据请使用 Secret。
- 单个 ConfigMap 的数据量上限为 1MB。

---

### 4.2 Secret 管理

#### 操作路径

左侧菜单 → **工作台** → **配置管理** → **Secret**

#### 概述

Secret 用于存储敏感数据，如密码、Token、证书、密钥等。Secret 中的数据以 Base64 编码存储，在使用时自动解码。

#### 创建 Secret

1. 在 Secret 列表页面，点击 **创建 Secret**。
2. 选择 Secret 类型：

| 类型 | 说明 |
|------|------|
| Opaque | 通用类型，存储任意键值对 |
| kubernetes.io/dockerconfigjson | 镜像仓库认证凭证 |
| kubernetes.io/tls | TLS 证书和私钥 |
| kubernetes.io/basic-auth | 用户名密码认证 |

3. 填写基本信息（名称、所属项目、描述）。
4. 根据类型填写数据：
   - **Opaque**：输入键值对，值会自动 Base64 编码。
   - **镜像仓库凭证**：输入仓库地址、用户名、密码。
   - **TLS**：上传或粘贴证书和私钥内容。
   - **用户名密码**：输入用户名和密码。

5. 点击 **创建** 完成。

#### 在工作负载中引用 Secret

引用方式与 ConfigMap 类似，支持作为环境变量和挂载文件两种方式。

#### 注意事项

- Secret 虽然以 Base64 编码存储，但 Base64 并非加密，请结合 RBAC 权限控制保护 Secret 访问。
- 镜像拉取凭证（ImagePullSecret）需要在工作负载中显式关联才能生效。
- 修改 Secret 后，引用该 Secret 的工作负载可能需要重启才能获取最新值。
- 请勿在 Secret 中存储超大文件，单个 Secret 数据量上限为 1MB。

---

### 4.3 环境变量管理

#### 操作路径

左侧菜单 → **工作台** → **应用工作负载** → 选择工作负载 → **编辑** → **容器配置** → **环境变量**

#### 环境变量来源

平台支持以下几种环境变量设置方式：

| 来源 | 说明 | 适用场景 |
|------|------|----------|
| 直接输入 | 手动输入键值对 | 简单配置项、不常变更的值 |
| ConfigMap 引用 | 引用 ConfigMap 中的某个 Key | 非敏感配置的集中管理 |
| Secret 引用 | 引用 Secret 中的某个 Key | 密码、Token 等敏感配置 |
| Pod 字段引用 | 引用 Pod 的元数据（如 Pod 名称、IP） | 需要运行时信息的场景 |
| 资源字段引用 | 引用容器的资源限制值 | 需要感知资源配置的应用 |

#### 步骤说明

1. 在工作负载的容器配置中，找到 **环境变量** 区域。
2. 点击 **添加环境变量**。
3. 输入变量名称（Key）。
4. 选择变量来源并填写对应值。
5. 重复上述步骤添加更多变量。
6. 保存工作负载配置。

#### 注意事项

- 环境变量名称建议使用大写字母和下划线（如 `DATABASE_HOST`）。
- 同名环境变量，后定义的会覆盖先定义的。
- 引用 ConfigMap 或 Secret 时，确保被引用的资源已存在且在同一项目中。
- 修改环境变量后需要重启 Pod 才能生效（滚动更新会自动重启）。

## 5. 监控查看

### 5.1 监控仪表盘

#### 操作路径

左侧菜单 → **监控平台** → **监控仪表盘**

#### 步骤说明

1. 进入 **监控平台** → **监控仪表盘** 页面。
2. 选择监控维度：

| 维度 | 说明 |
|------|------|
| 集群监控 | 查看集群级别的资源使用和健康状态 |
| 节点监控 | 查看单个节点的 CPU、内存、磁盘、网络等指标 |
| 工作负载监控 | 查看应用的 Pod 资源使用和运行状态 |

3. 选择目标集群和时间范围。
4. 仪表盘展示以下核心指标：

**集群级指标：**
- CPU 使用率 / 请求率 / 总量
- 内存使用率 / 请求率 / 总量
- Pod 数量和状态分布
- 节点状态统计

**节点级指标：**
- CPU 使用率趋势图
- 内存使用率趋势图
- 磁盘 I/O 读写速率
- 网络流量（发送/接收）
- 文件系统使用率

**工作负载级指标：**
- Pod CPU / 内存使用
- Pod 重启次数
- 容器状态

5. 支持自定义时间范围查看历史数据。
6. 图表支持缩放和数据点 Hover 查看详细值。

#### 注意事项

- 监控数据基于 Prometheus 采集，默认采集间隔为 30 秒。
- 监控数据默认保留 15 天，超出保留期的数据会被自动清理。
- 如部分指标无数据，请检查目标集群的监控组件是否正常运行。

### 5.2 指标查询

#### 操作路径

左侧菜单 → **监控平台** → **指标查询**

#### 步骤说明

1. 进入 **指标查询** 页面。
2. 选择目标集群。
3. 输入 PromQL 查询表达式或从指标列表中选择：

**常用查询示例：**

| 查询目的 | PromQL 表达式 |
|----------|---------------|
| 节点 CPU 使用率 | `100 - (avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)` |
| 节点内存使用率 | `(1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100` |
| Pod CPU 使用 | `sum(rate(container_cpu_usage_seconds_total{namespace="<ns>"}[5m])) by (pod)` |
| Pod 内存使用 | `sum(container_memory_working_set_bytes{namespace="<ns>"}) by (pod)` |

4. 设置查询时间范围和步长。
5. 点击 **查询** 查看结果。
6. 结果以图表和表格两种形式展示。

#### 注意事项

- PromQL 查询可能消耗较多资源，建议合理设置查询时间范围，避免查询过长时间段。
- 自定义查询时注意 Label 的准确性，错误的 Label 会导致查询无结果。
- 常用查询可以保存为模板，方便后续复用。

### 5.3 告警查看

#### 操作路径

左侧菜单 → **监控平台** → **告警管理** → **告警列表**

#### 步骤说明

1. 进入 **告警管理** → **告警列表** 页面。
2. 页面展示当前活跃的告警列表：

| 字段 | 说明 |
|------|------|
| 告警名称 | 触发的告警规则名称 |
| 告警级别 | Critical / Warning / Info |
| 告警状态 | Firing（触发中）/ Resolved（已恢复） |
| 触发时间 | 告警首次触发的时间 |
| 持续时间 | 告警持续的时间长度 |
| 告警来源 | 触发告警的集群和资源 |

3. 支持按告警级别、状态和时间范围筛选。
4. 点击告警名称查看详情，包括：
   - 告警规则的查询表达式。
   - 触发阈值和当前值。
   - 告警的历史触发记录。
   - 关联的资源和 Label 信息。

#### 告警级别说明

| 级别 | 含义 | 处理建议 |
|------|------|----------|
| **Critical** | 严重告警，影响业务运行 | 立即处理 |
| **Warning** | 警告告警，可能影响业务 | 尽快关注和处理 |
| **Info** | 信息告警，用于提示 | 知悉即可，按需处理 |

#### 注意事项

- 告警通知依赖于通知渠道配置（邮件、钉钉、企业微信等），如未收到通知请联系管理员检查配置。
- 告警恢复后状态会自动变更为 Resolved，但通知渠道的恢复通知需要单独配置。
- 频繁触发的告警建议与管理员沟通调整阈值或根因排查。

## 6. OTA 远程运维

### 6.1 远程命令

#### 操作路径

左侧菜单 → **OTA 管理** → **节点列表** → 选择目标节点 → **远程命令**

#### 概述

OTA（Over-The-Air）远程运维模块提供对边缘节点的远程管理能力，无需直接 SSH 登录边缘设备。通过平台可以远程执行命令、传输文件和编排批量任务。

#### 节点接入

使用 OTA 远程运维功能前，需要先将边缘节点接入 OTA 管理：

1. 进入 **OTA 管理** → **节点列表**。
2. 点击 **接入节点**。
3. 系统生成接入脚本，包含节点注册的 Agent 安装命令。
4. 在目标边缘节点上执行该脚本，安装 OTA Agent。
5. Agent 安装成功后，节点状态变为 **在线**。

#### 执行远程命令

1. 在 OTA 节点列表中，勾选一个或多个目标节点。
2. 点击 **远程命令** 按钮。
3. 输入要执行的 Shell 命令。
4. 配置执行参数：

| 参数 | 说明 |
|------|------|
| 超时时间 | 命令执行的最大等待时间（秒） |
| 执行用户 | 命令执行的系统用户（默认 root） |
| 并发数 | 同时执行命令的节点数量 |

5. 点击 **执行**。
6. 查看执行结果：
   - 每个节点的执行状态（成功/失败/超时）。
   - 命令的标准输出和错误输出。
   - 执行耗时。

#### 注意事项

- 远程命令以指定用户身份在目标节点上执行，请注意命令的安全性。
- 批量执行命令时，建议先在少量节点上测试验证。
- 命令执行超时后会被强制终止，请合理设置超时时间。
- 所有远程命令执行记录会被记入审计日志。

### 6.2 文件传输

#### 操作路径

左侧菜单 → **OTA 管理** → **节点列表** → 选择目标节点 → **文件传输**

#### 步骤说明

**上传文件到边缘节点：**

1. 在 OTA 节点列表中，勾选目标节点。
2. 点击 **文件传输** → **上传文件**。
3. 配置上传参数：

| 参数 | 说明 |
|------|------|
| 源文件 | 从本地选择要上传的文件 |
| 目标路径 | 边缘节点上的目标存放路径 |
| 文件权限 | 上传后的文件权限（如 644、755） |
| 文件属主 | 文件的所属用户和用户组 |

4. 点击 **开始传输**。
5. 查看传输进度和结果。

**从边缘节点下载文件：**

1. 在 OTA 节点列表中，选择目标节点。
2. 点击 **文件传输** → **下载文件**。
3. 输入边缘节点上的文件路径。
4. 点击 **下载**，文件保存到本地。

#### 注意事项

- 单次文件传输大小限制为 100MB，超大文件建议分片传输或使用其他方式。
- 文件传输通过加密通道进行，确保数据安全。
- 批量上传文件时，会依次传输到每个节点，请关注传输进度。
- 上传文件会覆盖目标路径下的同名文件，请注意备份。

### 6.3 Playbook 编排

#### 操作路径

左侧菜单 → **OTA 管理** → **任务管理** → **创建任务**

#### 概述

Playbook 编排功能允许将多个运维操作组合为一个有序的任务流程，实现复杂运维场景的自动化执行。支持命令执行、文件传输、脚本运行等操作的编排和调度。

#### 创建 Playbook 任务

1. 进入 **OTA 管理** → **任务管理** 页面。
2. 点击 **创建任务**。
3. 填写任务基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 任务名称 | 是 | 任务的名称标识 |
| 任务描述 | 否 | 任务用途和说明 |
| 目标节点 | 是 | 选择执行任务的目标节点（支持多选） |

4. 编排任务步骤：
   - 点击 **添加步骤**。
   - 选择步骤类型：
     - **执行命令**：在目标节点上执行 Shell 命令。
     - **上传文件**：将文件传输到目标节点。
     - **执行脚本**：上传并执行脚本文件。
   - 为每个步骤配置参数和执行条件。
   - 拖拽调整步骤执行顺序。

5. 配置执行策略：

| 策略 | 说明 |
|------|------|
| 执行模式 | 串行（逐节点执行）/ 并行（同时执行） |
| 失败策略 | 忽略失败继续 / 失败即停止 |
| 并发数 | 并行模式下的最大并发节点数 |
| 超时时间 | 单个步骤的最大执行时间 |

6. 点击 **保存** 保存任务（不立即执行），或点击 **保存并执行** 立即执行。

#### 查看任务执行结果

1. 在 **任务管理** 列表中，点击目标任务名称。
2. 查看任务执行概况：
   - 整体执行状态和进度。
   - 各步骤的执行状态。
   - 各节点的执行结果。
3. 点击具体节点或步骤查看详细的输出日志。

#### 注意事项

- 编排复杂任务时建议添加步骤间的检查点，确认上一步成功再执行下一步。
- 生产环境建议使用"失败即停止"策略，避免错误操作扩散。
- 已保存的任务可以多次执行，适合重复性运维场景。
- 所有任务执行记录和日志会保留在系统中，可供后续审计和问题排查。
