## 1. 用户管理

### 1.1 用户列表

#### 操作路径

左侧菜单 → **IAM 管理** → **用户管理** → **用户列表**

#### 步骤说明

1. 点击左侧菜单中的 **IAM 管理**，展开子菜单。
2. 点击 **用户管理**，进入用户列表页面。
3. 页面以表格形式展示平台所有用户账号信息：

| 字段 | 说明 |
|------|------|
| 用户名 | 用户的唯一登录标识 |
| 显示名称 | 用户的显示名称（姓名） |
| 邮箱 | 用户的电子邮箱 |
| 状态 | 启用 / 禁用 / 锁定 |
| 角色 | 用户绑定的角色列表 |
| 最近登录 | 用户最后一次登录的时间 |
| 创建时间 | 账号创建时间 |

4. 支持按用户名、邮箱搜索。
5. 支持按状态（启用/禁用/锁定）筛选。
6. 点击用户名可进入用户详情页，查看详细信息和操作记录。

#### 用户详情页

用户详情页包含以下信息：

- **基本信息**：用户名、显示名称、邮箱、手机号、创建时间。
- **角色绑定**：用户关联的所有角色及其作用范围。
- **登录历史**：最近的登录记录（时间、IP 地址、登录方式、结果）。
- **操作记录**：用户在平台上的操作审计记录。

#### 注意事项

- 用户列表仅对具有 IAM 管理权限的管理员可见。
- 锁定状态的用户无法登录，需要管理员手动解锁或等待自动解锁。
- 禁用用户后，该用户当前的活跃会话不会立即中断，需等待会话过期。

### 1.2 创建用户

#### 操作路径

左侧菜单 → **IAM 管理** → **用户管理** → 点击 **创建用户**

#### 步骤说明

1. 在用户列表页面，点击右上角 **创建用户** 按钮。
2. 填写用户基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 用户名 | 是 | 2-32 个字符，仅支持小写字母、数字和连字符，以字母开头 |
| 显示名称 | 是 | 用户的真实姓名或显示名 |
| 邮箱 | 是 | 用户的电子邮箱，需要唯一 |
| 手机号 | 否 | 用户的手机号码 |
| 初始密码 | 是 | 设置初始登录密码，需满足密码策略 |
| 确认密码 | 是 | 再次输入密码确认一致 |

3. 配置用户选项：
   - **首次登录强制修改密码**：勾选后用户首次登录必须修改密码（建议开启）。
   - **密码过期时间**：设置密码有效期（默认跟随全局密码策略）。

4. 分配角色（可选，也可后续在角色绑定中操作）：
   - 点击 **添加角色**。
   - 选择要分配的角色。
   - 设置角色的作用范围（全局 / 指定工作空间 / 指定集群）。

5. 点击 **创建** 完成用户创建。

#### 批量创建用户

1. 在用户列表页面，点击 **批量导入**。
2. 下载用户导入模板（CSV 格式）。
3. 按模板格式填写用户信息。
4. 上传 CSV 文件。
5. 系统校验数据格式，展示校验结果。
6. 确认后批量创建用户。

#### 用户操作

| 操作 | 说明 |
|------|------|
| **编辑** | 修改用户的基本信息（显示名称、邮箱、手机号等） |
| **重置密码** | 为用户重置登录密码，新密码需满足密码策略 |
| **禁用/启用** | 禁用或启用用户账号 |
| **解锁** | 解除因密码错误次数过多导致的账号锁定 |
| **删除** | 删除用户账号（不可逆操作） |

#### 注意事项

- 用户名创建后不可修改。
- 删除用户前需先解除该用户的所有角色绑定。
- 批量导入时，密码字段为空则使用默认密码，建议导入后通知用户修改密码。
- 建议所有新用户均开启"首次登录强制修改密码"选项。

### 1.3 密码策略

#### 操作路径

左侧菜单 → **系统配置** → **安全设置** → **密码策略**

#### 步骤说明

1. 进入 **系统配置** → **安全设置** → **密码策略** 页面。
2. 配置密码策略参数：

| 策略项 | 默认值 | 说明 |
|--------|--------|------|
| 最小密码长度 | 8 | 密码的最少字符数 |
| 必须包含大写字母 | 是 | 至少包含一个大写字母 |
| 必须包含小写字母 | 是 | 至少包含一个小写字母 |
| 必须包含数字 | 是 | 至少包含一个数字 |
| 必须包含特殊字符 | 是 | 至少包含一个特殊字符（如 !@#$%） |
| 密码有效期（天） | 90 | 密码过期后强制修改，0 表示永不过期 |
| 密码重复限制 | 5 | 新密码不能与最近 N 次使用过的密码相同 |
| 登录失败锁定次数 | 5 | 连续登录失败 N 次后锁定账号 |
| 账号锁定时间（分钟） | 30 | 锁定后的自动解锁等待时间 |
| 会话超时时间（分钟） | 30 | 无操作超过该时间后会话自动过期 |

3. 修改需要调整的策略项。
4. 点击 **保存** 使配置生效。

#### 注意事项

- 密码策略修改后仅对新设置的密码生效，不会强制已有用户立即修改密码。
- 如需强制所有用户按新策略更新密码，可以设置密码有效期并等待密码过期。
- 密码策略应在安全性和可用性之间取得平衡，过于复杂的策略可能导致用户频繁忘记密码。
- 建议生产环境开启所有复杂度要求，并设置合理的密码有效期。

## 2. 角色与权限管理

### 2.1 内置角色

#### 操作路径

左侧菜单 → **IAM 管理** → **角色管理**

#### 步骤说明

1. 进入 **角色管理** 页面。
2. 系统提供以下内置角色：

| 角色名称 | 作用范围 | 说明 |
|----------|----------|------|
| **platform-admin** | 全局 | 平台管理员，拥有平台所有功能的完全访问权限 |
| **cluster-admin** | 集群 | 集群管理员，拥有指定集群的完全管理权限 |
| **workspace-admin** | 工作空间 | 工作空间管理员，管理工作空间下的项目、成员和资源 |
| **project-admin** | 项目 | 项目管理员，管理指定项目下的所有资源 |
| **developer** | 项目 | 应用开发者，可以在项目中部署和管理应用 |
| **viewer** | 项目 | 只读用户，仅可查看项目下的资源信息 |

3. 点击角色名称查看角色详情：
   - **权限列表**：该角色包含的所有权限项。
   - **关联用户**：已绑定该角色的用户列表。
   - **权限模板**：角色引用的权限模板。

#### 内置角色权限矩阵

| 功能模块 | platform-admin | cluster-admin | workspace-admin | developer | viewer |
|----------|:-:|:-:|:-:|:-:|:-:|
| 用户管理 | ✓ | ✗ | ✗ | ✗ | ✗ |
| 角色管理 | ✓ | ✗ | ✗ | ✗ | ✗ |
| 集群创建/删除 | ✓ | ✗ | ✗ | ✗ | ✗ |
| 集群管理 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 节点管理 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 工作空间管理 | ✓ | ✗ | ✓ | ✗ | ✗ |
| 项目管理 | ✓ | ✗ | ✓ | ✗ | ✗ |
| 应用部署 | ✓ | ✓ | ✓ | ✓ | ✗ |
| 应用查看 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 配置管理 | ✓ | ✓ | ✓ | ✓ | ✗ |
| 监控查看 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 告警管理 | ✓ | ✓ | ✗ | ✗ | ✗ |
| OTA 管理 | ✓ | ✓ | ✗ | ✗ | ✗ |
| 审计日志 | ✓ | ✗ | ✗ | ✗ | ✗ |
| 系统配置 | ✓ | ✗ | ✗ | ✗ | ✗ |

#### 注意事项

- 内置角色不可删除和修改，如需自定义权限请创建自定义角色。
- `platform-admin` 角色拥有平台所有权限，应严格限制分配人数。
- 建议遵循最小权限原则，仅授予用户完成工作所需的最小权限集。

### 2.2 自定义角色

#### 操作路径

左侧菜单 → **IAM 管理** → **角色管理** → 点击 **创建角色**

#### 步骤说明

1. 在角色管理页面，点击 **创建角色** 按钮。
2. 填写角色基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 角色名称 | 是 | 角色的唯一标识名称 |
| 显示名称 | 是 | 角色的中文显示名 |
| 描述 | 否 | 角色的用途和权限范围说明 |
| 作用范围 | 是 | 全局 / 集群 / 工作空间 / 项目 |

3. 配置权限：

   **方式一：基于权限模板**
   - 选择一个或多个预定义的权限模板。
   - 权限模板包含一组常用权限的集合，可快速组合。

   **方式二：自定义权限项**
   - 按功能模块勾选具体的权限项。
   - 权限按模块分组展示（集群管理、应用管理、配置管理等）。
   - 每个权限项包含操作类型：查看（Get/List）、创建（Create）、更新（Update）、删除（Delete）。

4. 点击 **创建** 完成角色创建。

#### 编辑自定义角色

1. 在角色列表中找到目标角色。
2. 点击操作列的 **编辑**。
3. 修改角色信息和权限配置。
4. 点击 **保存**。

#### 删除自定义角色

1. 点击操作列的 **删除**。
2. 系统检查是否有用户绑定了该角色。
3. 如有绑定的用户，需先解除绑定关系才能删除。
4. 确认删除。

#### 注意事项

- 自定义角色的作用范围创建后不可修改。
- 修改角色权限后，已绑定该角色的用户权限会立即生效。
- 建议在创建自定义角色前，先梳理目标用户群的职责和所需操作。
- 角色名称创建后不可修改，请谨慎命名。

---

### 2.3 角色绑定

#### 操作路径

**方式一：** 左侧菜单 → **IAM 管理** → **用户管理** → 选择用户 → **角色绑定** Tab

**方式二：** 左侧菜单 → **IAM 管理** → **角色管理** → 选择角色 → **关联用户** Tab

#### 步骤说明

**为用户分配角色：**

1. 进入用户详情页 → **角色绑定** Tab。
2. 点击 **添加角色绑定**。
3. 选择要绑定的角色。
4. 设置角色的作用范围：

| 作用范围 | 说明 |
|----------|------|
| 全局 | 角色在整个平台范围内生效 |
| 指定集群 | 角色仅在选定的集群中生效 |
| 指定工作空间 | 角色仅在选定的工作空间中生效 |
| 指定项目 | 角色仅在选定的项目中生效 |

5. 点击 **确认** 完成绑定。

**为角色批量关联用户：**

1. 进入角色详情页 → **关联用户** Tab。
2. 点击 **关联用户**。
3. 从用户列表中勾选要关联的用户。
4. 设置统一的作用范围。
5. 点击 **确认** 完成关联。

#### 解除角色绑定

1. 在用户的角色绑定列表或角色的关联用户列表中。
2. 找到要解除的绑定关系。
3. 点击 **解除绑定**。
4. 确认后，权限立即生效。

#### 注意事项

- 一个用户可以绑定多个角色，最终权限为所有角色权限的并集。
- 角色绑定和解除绑定即时生效，无需用户重新登录。
- 解除角色绑定后，该用户将立即失去对应权限，正在进行的操作可能受影响。
- 建议定期审查角色绑定关系，清理不再需要的权限分配。

## 3. 资源配额管理

### 3.1 配额层级

#### 概述

资源配额用于限制不同层级的资源使用量，防止资源过度使用和确保公平分配。平台支持多层级的资源配额管理体系：

```
平台总资源
├── 集群配额        ← 限制单个集群的最大资源使用量
│   ├── 工作空间配额  ← 限制工作空间下所有项目的总资源使用量
│   │   ├── 项目配额  ← 限制单个项目（Namespace）的资源使用量
│   │   └── 项目配额
│   └── 工作空间配额
└── 集群配额
```

#### 可管理的资源类型

| 资源类型 | 说明 | 单位 |
|----------|------|------|
| CPU | 处理器计算资源 | 核（Core） |
| 内存 | 内存资源 | GiB |
| 存储 | 持久化存储资源 | GiB |
| Pod 数量 | 可创建的 Pod 最大数量 | 个 |
| GPU | GPU 设备资源 | 个 / 显存 |

---

### 3.2 集群配额

#### 操作路径

左侧菜单 → **集群管理** → 选择集群 → **集群设置** → **资源配额**

#### 步骤说明

1. 进入集群详情页 → **集群设置** Tab。
2. 点击 **资源配额** 选项。
3. 查看当前集群的资源总量和已使用量。
4. 设置集群级别的资源配额上限：

| 字段 | 说明 |
|------|------|
| CPU 配额 | 集群可分配的最大 CPU 核数 |
| 内存配额 | 集群可分配的最大内存量 |
| 存储配额 | 集群可分配的最大存储量 |
| Pod 数量上限 | 集群可创建的最大 Pod 数量 |
| GPU 配额 | 集群可分配的最大 GPU 数量（如适用） |

5. 点击 **保存** 使配置生效。

#### 注意事项

- 集群配额不能超过集群实际物理资源总量。
- 修改集群配额不会影响已运行的工作负载，但会限制新的资源申请。
- 当集群资源使用接近配额时，新的 Pod 创建可能因资源不足而被拒绝。
- 建议集群配额预留 10%-20% 的余量，用于系统组件和突发需求。

---

### 3.3 工作空间 / 项目配额

#### 操作路径

**工作空间配额：** 左侧菜单 → **工作空间管理** → 选择工作空间 → **配额管理**

**项目配额：** 工作空间 → 选择项目 → **项目设置** → **资源配额**

#### 设置工作空间配额

1. 进入工作空间管理页面，选择目标工作空间。
2. 点击 **配额管理** Tab。
3. 设置工作空间的资源配额：

| 字段 | 说明 |
|------|------|
| CPU 配额 | 工作空间可使用的最大 CPU 核数 |
| 内存配额 | 工作空间可使用的最大内存量 |
| 存储配额 | 工作空间可使用的最大存储量 |
| 项目数量限制 | 工作空间下可创建的最大项目数 |

4. 点击 **保存**。

#### 设置项目配额

1. 进入项目设置页面。
2. 点击 **资源配额** 选项。
3. 设置项目的资源配额：

| 字段 | 说明 |
|------|------|
| CPU Request 配额 | 项目下所有 Pod 的 CPU Request 总和上限 |
| CPU Limit 配额 | 项目下所有 Pod 的 CPU Limit 总和上限 |
| 内存 Request 配额 | 项目下所有 Pod 的内存 Request 总和上限 |
| 内存 Limit 配额 | 项目下所有 Pod 的内存 Limit 总和上限 |
| Pod 数量上限 | 项目中可创建的最大 Pod 数量 |
| PVC 数量上限 | 项目中可创建的最大 PersistentVolumeClaim 数量 |
| 存储配额 | 项目可申请的存储总量 |

4. 点击 **保存** 使配置生效。

#### 查看配额使用情况

1. 在工作空间或项目的配额管理页面。
2. 查看资源使用情况的可视化展示：
   - **使用量 / 配额量**：当前已使用资源与配额的对比。
   - **使用率**：资源使用百分比进度条。
   - **趋势图**：资源使用量的历史趋势。

#### 注意事项

- 项目配额之和不能超过所属工作空间的配额。
- 工作空间配额之和不能超过所属集群的配额。
- 配额变更即时生效，但不会终止已运行的超额工作负载。
- 建议为每个项目都设置合理的配额，避免单个项目消耗过多资源影响其他项目。
- 未设置配额的项目将共享工作空间的剩余配额，可能导致资源抢占。

## 4. 审计日志

### 4.1 操作日志

#### 操作路径

左侧菜单 → **系统配置** → **审计日志** → **操作日志**

#### 步骤说明

1. 进入 **审计日志** → **操作日志** 页面。
2. 页面展示平台所有用户的操作记录：

| 字段 | 说明 |
|------|------|
| 操作时间 | 操作发生的时间 |
| 操作者 | 执行操作的用户名 |
| 操作类型 | Create / Update / Delete / Patch 等 |
| 资源类型 | 操作的目标资源类型（Cluster / Deployment / User 等） |
| 资源名称 | 操作的目标资源名称 |
| 操作结果 | 成功 / 失败 |
| 来源 IP | 操作者的客户端 IP 地址 |
| 操作详情 | 操作的详细请求和响应信息 |

3. 支持多维度筛选：
   - **时间范围**：选择查询的时间段。
   - **操作者**：按用户筛选。
   - **操作类型**：按 Create / Update / Delete 等筛选。
   - **资源类型**：按资源类型筛选。
   - **操作结果**：按成功/失败筛选。

4. 点击某条记录可展开查看详细信息，包括请求参数和响应内容。

#### 导出操作日志

1. 配置筛选条件。
2. 点击 **导出** 按钮。
3. 选择导出格式（CSV / Excel）。
4. 下载导出文件。

#### 注意事项

- 操作日志默认保留 180 天，超出保留期的日志会被自动清理。
- 操作日志为只读数据，不可修改或删除，确保审计追溯的可靠性。
- 大范围时间段的日志查询可能耗时较长，建议缩小查询范围。
- 建议定期导出重要时期的操作日志进行归档。

### 4.2 登录日志

#### 操作路径

左侧菜单 → **系统配置** → **审计日志** → **登录日志**

#### 步骤说明

1. 进入 **审计日志** → **登录日志** 页面。
2. 页面展示所有用户的登录和登出记录：

| 字段 | 说明 |
|------|------|
| 时间 | 登录/登出的时间 |
| 用户名 | 登录的用户名 |
| 登录方式 | 密码登录 / SSO / LDAP / OAuth2 |
| 事件类型 | 登录成功 / 登录失败 / 退出登录 / 会话过期 |
| 来源 IP | 客户端 IP 地址 |
| 浏览器/客户端 | 浏览器 User-Agent 信息 |
| 失败原因 | 登录失败时的原因说明（密码错误、账号锁定等） |

3. 支持筛选：
   - **时间范围**：查询指定时间段的记录。
   - **用户名**：查询指定用户的登录记录。
   - **事件类型**：按登录成功/失败等筛选。
   - **来源 IP**：查询指定 IP 的登录记录。

#### 安全分析

通过登录日志可以发现以下安全异常：

- **异常登录时间**：非工作时间段的登录活动。
- **异常登录 IP**：来自非常用 IP 地址的登录。
- **频繁登录失败**：可能存在暴力破解攻击。
- **多地同时登录**：同一账号从不同 IP 同时登录。

#### 注意事项

- 登录日志默认保留 365 天。
- 登录失败记录对于安全审计尤为重要，应重点关注。
- 如发现异常登录行为，应立即检查相关账号的安全状态，必要时禁用或重置密码。
- 建议定期审查登录日志，确保平台安全。

## 5. 告警规则配置

### 5.1 创建告警规则

#### 操作路径

左侧菜单 → **监控平台** → **告警管理** → **告警规则** → 点击 **创建规则**

#### 步骤说明

1. 进入 **告警管理** → **告警规则** 页面。
2. 点击 **创建规则** 按钮。
3. 填写规则基本信息：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 规则名称 | 是 | 告警规则的名称标识 |
| 规则描述 | 否 | 规则的用途和触发说明 |
| 告警级别 | 是 | Critical / Warning / Info |
| 所属集群 | 是 | 规则监控的目标集群 |
| 规则分组 | 否 | 将规则归入分组，便于管理 |

4. 配置告警条件：

   **方式一：预设指标**
   - 从预设指标列表中选择监控对象和指标：
     - 节点 CPU 使用率
     - 节点内存使用率
     - 节点磁盘使用率
     - Pod 重启次数
     - Pod 异常状态
     - 其他常用指标
   - 设置阈值和比较条件（大于/小于/等于）。

   **方式二：自定义 PromQL**
   - 输入自定义的 PromQL 查询表达式。
   - 设置触发条件（表达式结果大于/小于指定阈值）。

5. 配置触发参数：

| 参数 | 说明 |
|------|------|
| 持续时间 | 条件持续满足多长时间后触发告警（如 5 分钟） |
| 评估间隔 | 规则的评估周期（如每 1 分钟评估一次） |

6. 配置通知：
   - 选择通知渠道（详见 5.2 通知渠道配置）。
   - 设置通知内容模板（可选）。
   - 配置重复通知间隔（告警持续时的通知频率）。

7. 点击 **创建** 完成。

#### 常用告警规则模板

| 规则名称 | PromQL 示例 | 阈值 | 级别 |
|----------|-------------|------|------|
| 节点 CPU 使用率过高 | `100 - avg by(instance)(rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100 > 90` | >90% 持续 5 分钟 | Warning |
| 节点内存不足 | `(1 - node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes) * 100 > 95` | >95% 持续 5 分钟 | Critical |
| 节点磁盘空间不足 | `(1 - node_filesystem_avail_bytes / node_filesystem_size_bytes) * 100 > 85` | >85% 持续 10 分钟 | Warning |
| Pod 频繁重启 | `increase(kube_pod_container_status_restarts_total[1h]) > 5` | >5 次/小时 | Warning |
| 节点不可达 | `kube_node_status_condition{condition="Ready",status="true"} == 0` | 持续 3 分钟 | Critical |

#### 管理告警规则

| 操作 | 说明 |
|------|------|
| **编辑** | 修改规则的条件、阈值、通知配置等 |
| **启用/禁用** | 临时启用或禁用规则，禁用后不再触发告警 |
| **删除** | 删除告警规则 |
| **复制** | 基于现有规则快速创建新规则 |

#### 注意事项

- 告警规则的评估间隔不宜设置过短，建议不低于 30 秒。
- 持续时间设置过短可能导致瞬时波动触发误告警，建议至少 3-5 分钟。
- 自定义 PromQL 需确保语法正确，建议先在指标查询页面验证。
- 创建过多告警规则可能影响监控系统性能，建议控制规则总数。

### 5.3 告警静默

#### 操作路径

左侧菜单 → **监控平台** → **告警管理** → **告警静默**

#### 概述

告警静默用于在特定时间段内临时抑制指定告警的通知发送。常用于计划维护窗口、已知问题处理期间等场景，避免产生大量重复通知干扰。

#### 创建静默规则

1. 进入 **告警管理** → **告警静默** 页面。
2. 点击 **创建静默规则**。
3. 配置静默规则：

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 规则名称 | 是 | 静默规则的名称 |
| 开始时间 | 是 | 静默生效的开始时间 |
| 结束时间 | 是 | 静默结束的时间（到期后自动恢复通知） |
| 创建者 | 自动 | 当前操作用户 |
| 备注 | 否 | 静默原因说明（如"计划维护"） |

4. 配置匹配条件（决定哪些告警被静默）：
   - **按告警名称**：指定要静默的告警规则名称。
   - **按告警级别**：静默指定级别的所有告警。
   - **按标签匹配**：通过 Label 匹配（如 `cluster=edge-01`、`namespace=production`）。
   - 支持多个匹配条件组合。

5. 点击 **创建** 完成。

#### 管理静默规则

| 操作 | 说明 |
|------|------|
| **查看** | 查看静默规则详情和匹配到的告警数量 |
| **编辑** | 修改静默的时间范围和匹配条件 |
| **提前结束** | 在到期前手动结束静默 |
| **删除** | 删除静默规则（等同于提前结束） |

#### 注意事项

- 告警静默只抑制通知发送，不影响告警的触发和记录。静默期间触发的告警仍然会被记录。
- 静默规则到期后会自动失效，无需手动恢复。
- 请在静默规则中填写清晰的备注，便于后续审计和追溯。
- 避免创建时间过长或匹配范围过大的静默规则，以免遗漏重要告警。
- 创建静默规则后，在告警列表中被静默的告警会显示"已静默"标记。

## 6. 系统配置

### 6.1 认证配置

#### 操作路径

左侧菜单 → **系统配置** → **认证配置**

#### 概述

平台支持多种身份认证方式，管理员可以根据企业需求配置外部认证源，实现与企业现有身份系统的集成。

#### 本地认证

平台默认使用本地认证方式，用户账号和密码存储在平台内部。本地认证无需额外配置，但管理员可以调整密码策略（见 1.3 密码策略）。

#### LDAP/AD 认证

1. 进入 **认证配置** 页面。
2. 点击 **LDAP** 选项卡 → **启用 LDAP 认证**。
3. 配置 LDAP 连接参数：

| 字段 | 说明 |
|------|------|
| LDAP 服务器地址 | LDAP 服务器的 URL（如 `ldap://ldap.example.com:389`） |
| 绑定 DN | 用于连接 LDAP 的管理员 DN |
| 绑定密码 | 管理员 DN 的密码 |
| 用户搜索基础 DN | 用户搜索的基础 DN（如 `ou=users,dc=example,dc=com`） |
| 用户搜索过滤器 | 用户搜索的 LDAP 过滤条件（如 `(uid=%s)`） |
| 用户名属性 | 映射为平台用户名的 LDAP 属性（如 `uid` 或 `sAMAccountName`） |
| 邮箱属性 | 映射为用户邮箱的 LDAP 属性（如 `mail`） |
| TLS/SSL | 是否启用加密连接 |

4. 点击 **测试连接** 验证 LDAP 连接是否正常。
5. 点击 **保存** 完成配置。

#### OAuth2/OIDC 认证

1. 进入 **认证配置** 页面。
2. 点击 **OAuth2** 选项卡 → **启用 OAuth2 认证**。
3. 配置 OAuth2 参数：

| 字段 | 说明 |
|------|------|
| Provider 名称 | 认证提供商名称（如"企业 SSO"） |
| Client ID | OAuth2 客户端 ID |
| Client Secret | OAuth2 客户端密钥 |
| Authorization URL | 授权端点 URL |
| Token URL | Token 获取端点 URL |
| UserInfo URL | 用户信息端点 URL |
| Scopes | 请求的权限范围（如 `openid profile email`） |
| 用户名映射字段 | 映射为平台用户名的字段 |

4. 点击 **测试** 验证 OAuth2 配置。
5. 点击 **保存** 完成配置。

#### 注意事项

- 启用外部认证后，用户可以使用本地账号或外部认证账号登录。
- LDAP 用户首次登录时会自动创建平台账号，后续登录使用 LDAP 密码验证。
- OAuth2 配置变更后需要重新登录才能生效。
- 建议在配置外部认证前先在测试环境验证。
- 外部认证服务不可用时，本地管理员账号仍可正常登录。

### 6.3 备份恢复

#### 操作路径

左侧菜单 → **系统配置** → **备份恢复**

#### 概述

备份恢复功能用于保护平台的关键配置数据和状态。支持手动和自动备份，确保在发生故障时能够快速恢复平台运行。

#### 备份范围

| 备份内容 | 说明 |
|----------|------|
| 平台配置 | 系统参数、认证配置、密码策略等 |
| 用户数据 | 用户账号、角色定义、角色绑定关系 |
| 集群配置 | 集群注册信息、节点组配置 |
| 告警规则 | 告警规则定义和通知渠道配置 |
| 资源配额 | 各层级的配额配置 |
| CRD 资源 | 平台自定义资源定义数据 |

#### 手动备份

1. 进入 **备份恢复** 页面。
2. 点击 **立即备份** 按钮。
3. 填写备份信息：

| 字段 | 说明 |
|------|------|
| 备份名称 | 备份的标识名称（自动附加时间戳） |
| 备份描述 | 本次备份的原因说明 |
| 备份范围 | 选择要备份的数据范围（全量/指定模块） |

4. 点击 **开始备份**。
5. 等待备份完成，查看备份状态和大小。

#### 自动备份

1. 在 **备份恢复** 页面，点击 **备份策略** Tab。
2. 启用自动备份。
3. 配置备份策略：

| 参数 | 说明 |
|------|------|
| 备份周期 | 每天 / 每周 / 每月 |
| 备份时间 | 执行备份的时间点（建议选择业务低峰期） |
| 保留份数 | 保留最近 N 份备份（超出的自动删除） |
| 存储位置 | 备份文件的存储位置（本地 / 远程存储） |

4. 点击 **保存** 启用自动备份策略。

#### 恢复操作

1. 在备份列表中找到要恢复的备份版本。
2. 点击 **恢复** 按钮。
3. 选择恢复范围：
   - **全量恢复**：恢复所有备份数据。
   - **指定模块恢复**：只恢复特定模块的数据。
4. 系统弹出确认提示，显示恢复操作的影响说明。
5. 输入 `CONFIRM` 确认执行恢复。
6. 等待恢复完成。
7. 恢复完成后，建议验证平台各功能是否正常运行。

#### 注意事项

- 恢复操作会覆盖当前平台数据，执行前务必确认恢复的必要性。
- 建议在执行恢复前先对当前状态做一次手动备份。
- 恢复操作期间平台可能短暂不可用，建议在维护窗口内执行。
- 备份文件包含敏感配置信息，请确保备份存储位置的安全性。
- 跨版本恢复（备份版本与当前平台版本不同）可能存在兼容性问题，请查阅版本发布说明。
- 建议定期测试备份恢复流程，验证备份的有效性。

### 常见问题

**Q：用户忘记密码怎么办？**

A：管理员在用户列表中找到该用户，点击 **重置密码** 为用户设置新密码，并通知用户登录后修改。

**Q：用户被锁定如何处理？**

A：方式一：等待锁定时间到期后自动解锁（默认 30 分钟）。方式二：管理员在用户列表中点击 **解锁** 手动解除锁定。

**Q：如何查看某个用户的所有操作记录？**

A：进入审计日志 → 操作日志，在操作者筛选中输入目标用户名进行查询。

**Q：告警通知没有收到怎么办？**

A：1) 检查通知渠道配置是否正确，使用测试功能验证。2) 检查是否有告警静默规则匹配了该告警。3) 检查通知渠道服务是否正常（邮件服务器、钉钉 Webhook 等）。

**Q：备份恢复后部分功能异常怎么办？**

A：1) 检查恢复日志中是否有错误信息。2) 确认备份版本与当前平台版本兼容。3) 尝试重启平台核心组件（edge-apiserver、edge-controller）。4) 如问题持续，联系技术支持。
