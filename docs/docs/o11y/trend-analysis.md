# 趋势分析

> **导航路径**: 集群 > 选择集群 > 监控告警 > 趋势图
> **访问地址**: `/boss/clusters/{集群ID}/monitoring`（点击「趋势图」标签页）
> **所需权限**: 平台管理员或集群管理员

## 功能说明

趋势分析页面以折线图形式展示集群 CPU 使用率和内存使用率的历史变化趋势。通过观察趋势图，您可以识别资源使用的高峰时段、判断资源是否持续增长、评估是否需要扩容。

## 页面概览

![趋势图](./images/cluster-trend-charts.png)

趋势图页面包含两张折线图：

| 图表 | 说明 |
|------|------|
| CPU 使用率趋势 | 蓝色折线，展示 CPU 使用率随时间的变化（单位：%） |
| 内存使用率趋势 | 绿色折线，展示内存使用率随时间的变化（单位：%） |

横轴为时间（默认展示当天 24 小时），纵轴为使用率百分比。

## 操作指南

### 操作一：查看趋势图

**操作步骤**

1. 在集群监控告警页面，点击 **趋势图** 标签页

2. 查看 CPU 使用率趋势图和内存使用率趋势图

3. 将鼠标悬停在折线上的数据点，查看该时间点的具体数值

**操作结果**

了解集群资源使用的历史变化规律。

### 操作二：分析资源使用模式

**操作步骤**

1. 观察 CPU 使用率趋势图，识别以下模式：

   | 模式 | 含义 | 建议 |
   |------|------|------|
   | 白天高、夜间低 | 业务有明显的日间高峰 | 正常模式，无需特别处理 |
   | 持续上升 | 资源消耗在增长 | 关注是否需要扩容或优化 |
   | 突然飙升 | 可能有异常负载 | 检查是否有异常进程或攻击 |
   | 持续平稳 | 系统运行稳定 | 正常模式 |

2. 同样方式分析内存使用率趋势图

**操作结果**

判断集群的资源使用是否健康，是否需要采取优化或扩容措施。

## 常见问题

### 问题：趋势图没有数据
**现象**：折线图为空，没有显示任何数据点
**原因**：监控数据采集时间不足，或当天尚未产生足够的数据点
**解决**：等待一段时间后刷新页面，确保监控服务已正常运行

### 问题：趋势图数据不连续
**现象**：折线图中间有断裂
**原因**：监控服务在该时段可能重启或网络中断，导致部分时间段数据缺失
**解决**：短暂的数据中断属于正常情况；如频繁出现，联系平台管理员检查监控服务稳定性
