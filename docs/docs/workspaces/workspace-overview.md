# 租户空间概览

## 概述

租户空间概览页面提供了租户空间的全面视图，包括资源使用情况、项目状态、成员信息和系统健康状态。通过概览页面，管理员可以快速了解租户空间的整体状况。

## 访问概览页面

### 导航路径

1. 进入租户空间管理页面
2. 选择要查看的租户空间
3. 在租户空间详情页面中点击 **"概览"**
4. 或直接通过URL访问：`/boss/clusters/{cluster}/workspaces/{workspace}/overview`

## 概览页面结构

概览页面通常包含以下主要部分：

### 1. 基本信息卡片

显示租户空间的基本信息：

- **租户空间名称**：唯一标识符和显示名称
- **描述信息**：租户空间的用途说明
- **所属集群**：租户空间所在的集群
- **创建时间**：租户空间的创建日期
- **管理员**：租户空间的管理员信息
- **状态**：当前运行状态（活跃/禁用）

### 2. 资源统计卡片

显示租户空间的资源使用情况：

- **项目总数**：租户空间中的项目数量
- **容器组总数**：所有项目中的Pod总数
- **服务总数**：暴露的服务数量
- **部署总数**：应用部署的数量
- **成员数量**：租户空间的成员总数

### 3. 实时资源用量

显示当前的资源使用情况：

#### CPU使用率
- 已使用CPU核心数 / 总CPU配额
- 使用百分比和趋势图
- 按项目分组的CPU使用情况

#### 内存使用率
- 已使用内存量 / 总内存配额
- 使用百分比和趋势图
- 按项目分组的内存使用情况

#### 磁盘使用率
- 已使用存储空间 / 总存储配额
- 使用百分比和趋势图
- 按项目分组的存储使用情况

#### GPU使用率（如可用）
- 已使用GPU核心数 / 总GPU配额
- 使用百分比和分配情况

### 4. 项目状态概览

显示租户空间内项目的状态分布：

- **运行中的项目**：正常工作的项目数量
- **有问题的项目**：出现错误或警告的项目数量
- **空项目**：没有部署任何资源的项目

### 5. 成员信息

显示租户空间的成员统计：

- **总成员数**：租户空间的成员总数
- **管理员数量**：具有管理员权限的成员数
- **普通成员数量**：普通用户成员数
- **最近活跃成员**：最近活动的成员列表

### 6. 资源配额统计

显示租户空间的资源配额和分配情况：

#### CPU配额
- **预留配额**：保证可用的CPU资源
- **上限配额**：最大可使用的CPU资源
- **已分配**：已分配给项目的CPU资源
- **使用率**：已分配资源占总配额的百分比

#### 内存配额
- **预留配额**：保证可用的内存资源
- **上限配额**：最大可使用的内存资源
- **已分配**：已分配给项目的内存资源
- **使用率**：已分配资源占总配额的百分比

#### 存储配额
- **预留配额**：保证可用的存储空间
- **上限配额**：最大可使用的存储空间
- **已分配**：已分配给项目的存储空间
- **使用率**：已分配资源占总配额的百分比

### 7. 活动时间线

显示最近的操作活动：

- 项目创建和删除
- 成员添加和移除
- 配置变更
- 资源分配变更
- 系统事件

### 8. 告警信息

显示租户空间的告警状态：

- **当前告警**：活跃的告警数量
- **告警级别**：严重、警告、信息
- **告警类型**：资源、性能、安全等
- **快速响应**：点击可查看告警详情

## 数据更新机制

### 实时更新

概览页面的数据会定期自动刷新：

- **统计数据**：每30秒更新一次
- **资源用量**：每15秒更新一次
- **告警信息**：实时推送更新
- **活动时间线**：事件触发时立即更新

### 手动刷新

用户可以点击页面上的刷新按钮手动更新数据：

- **全部刷新**：更新所有数据
- **部分刷新**：更新特定模块的数据

## 数据可视化

### 图表类型

概览页面使用多种图表展示数据：

1. **饼图**：显示资源分配比例
2. **柱状图**：显示项目间的资源对比
3. **折线图**：显示资源使用趋势
4. **进度条**：显示配额使用进度
5. **状态指示器**：显示系统和项目状态

### 交互功能

- **悬停显示详情**：鼠标悬停在图表上显示详细数据
- **点击跳转**：点击图表元素可跳转到相关详情页面
- **图表缩放**：支持放大和缩小图表查看详情
- **导出数据**：支持导出图表数据为CSV或图片

## 使用场景

### 日常监控

管理员每天查看概览页面，了解：

1. 租户空间整体运行状态
2. 资源使用是否正常
3. 是否有告警需要处理
4. 项目和成员的活动情况

### 容量规划

基于概览页面的数据进行：

1. 分析资源使用趋势
2. 预测未来资源需求
3. 规划资源扩容或缩减
4. 优化资源分配策略

### 问题诊断

当出现问题时，通过概览页面：

1. 快速定位问题范围
2. 查看影响的项目和资源
3. 分析相关联的事件和告警
4. 制定解决方案

### 定期审查

定期（每周/每月）审查：

1. 资源使用效率
2. 项目和成员的活跃度
3. 安全策略执行情况
4. 成本和性能优化机会

## 最佳实践

### 监控指标关注

重点关注以下指标：

1. **资源使用率**：CPU、内存、存储使用率应保持在合理范围内
2. **项目健康度**：确保所有项目状态正常
3. **告警响应时间**：及时处理告警，避免问题扩大
4. **成员活跃度**：定期清理不活跃的成员

### 告警管理

1. **设置合理的告警阈值**
   - 避免过于敏感导致告警疲劳
   - 避免过于宽松导致错过重要问题

2. **分级处理告警**
   - 严重告警立即处理
   - 警告告警及时处理
   - 信息告警定期查看

3. **告警分析和优化**
   - 分析告警根因
   - 优化配置减少误报
   - 建立告警处理流程

### 性能优化

1. **资源优化**
   - 识别资源使用异常的项目
   - 优化应用配置提高效率
   - 清理未使用的资源

2. **成本优化**
   - 分析资源成本分布
   - 优化资源分配策略
   - 使用合适的实例类型和配置

## 故障排除

### 常见问题

**Q: 概览页面数据不更新？**

A: 解决方法：
- 检查网络连接是否正常
- 刷新浏览器页面
- 清除浏览器缓存
- 联系平台管理员检查系统状态

**Q: 资源使用率显示异常？**

A: 排查步骤：
- 检查是否有项目资源泄漏
- 查看是否有异常的Pod占用资源
- 验证监控数据是否准确
- 检查配额设置是否合理

**Q: 无法查看某些数据？**

A: 可能原因：
- 用户权限不足
- 数据源服务不可用
- 浏览器兼容性问题
- 网络防火墙阻止
