## 快速开始

### 1. 了解租户空间

**租户空间**是边缘智能管理平台的多租户隔离单元，提供：

- **资源隔离**：独立的计算、存储、网络资源
- **用户隔离**：独立的用户和权限体系
- **策略隔离**：独立的访问控制和安全策略

### 2. 创建第一个租户空间

1. 登录边缘智能管理平台
2. 进入"控制台" → "租户空间管理"
3. 点击"创建租户空间"
4. 填写基本信息并确认创建

详细步骤请参考：[租户空间管理](./workspace-management.md)

### 3. 创建项目并部署应用

1. 进入租户空间详情页面
2. 点击"项目管理" → "创建项目"
3. 在项目中部署应用
4. 配置资源和服务

详细步骤请参考：[项目管理](./workspace-projects.md)

## 使用场景

### 按团队组织

为不同的团队创建独立的租户空间：

```
开发团队租户空间 (dev-team)
├── 前端项目 (frontend-app)
├── 后端项目 (backend-api)
└── 测试项目 (testing-service)

测试团队租户空间 (qa-team)
├── 自动化测试项目 (auto-test)
└── 性能测试项目 (perf-test)
```

### 按环境组织

为不同的环境创建独立的租户空间：

```
开发环境租户空间 (dev-env)
├── 开发项目 (development)
└── 调试项目 (debug)

测试环境租户空间 (test-env)
├── 功能测试项目 (feature-test)
└── 集成测试项目 (integration-test)

生产环境租户空间 (prod-env)
├── 生产应用项目 (production-app)
└── 监控项目 (monitoring)
```

### 按客户组织（多租户场景）

为不同的客户创建独立的租户空间：

```
客户A租户空间 (customer-a)
├── Web应用项目 (web-app)
└── API服务项目 (api-service)

客户B租户空间 (customer-b)
├── 数据处理项目 (data-processing)
└── 分析服务项目 (analytics-service)
```

## 核心概念

### 租户空间层级结构

```
平台 (Platform)
└── 集群 (Cluster)
    └── 租户空间 (Workspace)
        └── 项目 (Project)
            └── 应用部署 (Deployment)
                └── 容器组 (Pod)
```

### 资源配额管理

租户空间支持三级资源配额管理：

1. **集群级配额**：平台管理员设置集群总资源
2. **租户空间级配额**：为租户空间分配资源
3. **项目级配额**：为项目分配资源

### 权限管理体系

租户空间提供完整的权限管理：

1. **平台权限**：平台级别的权限控制
2. **租户空间权限**：租户空间级别的权限
3. **项目权限**：项目级别的权限
4. **资源权限**：资源级别的权限

## 最佳实践

### 命名规范

**租户空间命名**：
- 使用小写字母、数字和连字符
- 以字母开头和结尾
- 不超过63个字符
- 示例：`dev-team`, `prod-env`, `customer-a`

**项目命名**：
- 使用描述性的名称
- 包含应用或环境信息
- 示例：`web-frontend`, `api-backend`, `test-service`

### 资源规划

**开发环境**：
- CPU：每个项目 0.5-2 核
- 内存：每个项目 1-4 GiB
- 存储：每个项目 10-50 GB

**测试环境**：
- CPU：每个项目 1-4 核
- 内存：每个项目 2-8 GiB
- 存储：每个项目 20-100 GB

**生产环境**：
- CPU：每个项目 2-8 核
- 内存：每个项目 4-16 GiB
- 存储：每个项目 50-500 GB

### 权限管理

**角色分配建议**：
- 租户空间管理员：1-2人
- 项目管理员：每个项目1人
- 开发人员：按需分配
- 只读用户：按需分配

**权限管理原则**：
- 遵循最小权限原则
- 定期审查权限分配
- 及时回收不需要的权限
- 使用角色简化管理

### 安全建议

**网络安全**：
- 启用项目间网络隔离
- 配置网络策略限制访问
- 使用服务网格增强安全性
- 定期审计网络配置

**访问控制**：
- 启用多因素认证
- 定期更换密码
- 限制访问IP范围
- 监控异常访问行为

**数据安全**：
- 使用Secret管理敏感信息
- 启用加密传输
- 定期备份重要数据
- 建立灾难恢复机制

## 故障排除

### 常见问题

**Q: 无法创建租户空间？**

A: 检查以下几点：
- 确认有创建租户空间的权限
- 检查集群是否有足够资源
- 验证租户空间名称是否符合规范
- 联系平台管理员检查配置

**Q: 资源配额不足？**

A: 解决方法：
- 分析资源使用情况
- 优化应用配置减少资源消耗
- 申请增加租户空间配额
- 将部分项目迁移到其他租户空间

**Q: 网络连接问题？**

A: 排查步骤：
- 检查网络策略配置
- 验证服务发现配置
- 检查DNS解析
- 查看防火墙规则

**Q: 权限问题？**

A: 解决方法：
- 确认用户角色和权限
- 检查权限继承关系
- 联系租户空间管理员
- 重新登录系统刷新权限
