# 选择和管理工作台

> **导航路径**: 顶部导航栏 > 智能边缘平台
> **访问地址**: `/tenant`
> **所需权限**: 平台用户

## 功能说明

工作台列表页面展示您有权访问的所有工作台。您可以在此选择要管理的工作台，或按集群和关键词筛选。

## 页面概览

![工作台列表](./images/tenant-workspace-list.png)

页面顶部显示「租户空间」标题，下方为工作台卡片列表。每张卡片包含工作台的显示名称、ID、运行状态、所属集群和管理员。

## 操作指南

### 操作一：按集群筛选工作台

**操作步骤**

1. 在页面顶部左侧，点击 **全部集群** 下拉框

2. 在下拉列表中选择目标集群（如 `host` 或 `v-oy-001`）

**操作结果**

列表仅显示属于所选集群的工作台。选择「全部集群」恢复显示所有工作台。

### 操作二：搜索工作台

**操作步骤**

1. 在页面顶部右侧的搜索框中，输入工作台名称或 ID 的关键词

2. 输入时列表自动筛选匹配结果

**操作结果**

列表仅显示名称或 ID 包含搜索关键词的工作台。

### 操作三：进入工作台

**操作步骤**

1. 在列表中找到目标工作台卡片

2. 点击卡片上的 **查看详情** 按钮

**操作结果**

进入该工作台的概览页面，左侧显示侧边栏导航菜单。

### 操作四：查看工作台概览

**操作步骤**

1. 进入工作台后，点击左侧菜单的 **总览**

   ![工作台概览](./images/tenant-overview-dashboard.png)

2. 概览页面包含以下区域：

   | 区域 | 内容 |
   |------|------|
   | 物理资源 | CPU、内存、持久卷的使用量和趋势图表，可切换时间范围 |
   | 应用资源用量 | 部署、有状态副本集、守护进程集、任务、定时任务、持久卷声明、服务、应用路由、容器组的数量统计 |
   | 基本信息 | 提供商、工作空间状态、所在集群、项目数量、Edge Runtime、IoT 平台服务状态 |
   | 资源用量排行 | 按 CPU 或内存用量对容器组进行排名 |

### 操作五：切换工作台

**操作步骤**

1. 在工作台页面左上角，点击当前工作台名称（如「ZGH边缘盒子」）

2. 在弹出的下拉菜单中，选择要切换到的目标工作台

3. 如需返回工作台列表，选择下拉菜单中的「全部租户空间」

**操作结果**

页面自动跳转到目标工作台的概览页面。

## 常见问题

### 问题：找不到某个工作台
**现象**：在列表中看不到预期的工作台
**原因**：可能被集群筛选过滤，或您没有该工作台的访问权限
**解决**：将集群筛选设置为「全部集群」；如仍找不到，联系管理员确认您是否已被邀请

### 问题：工作台卡片信息不更新
**现象**：工作台状态或统计信息未及时更新
**原因**：页面数据有缓存
**解决**：点击搜索框旁边的刷新按钮，或刷新浏览器页面
