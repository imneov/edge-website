# 权限说明

> **所需权限**: 平台管理员

## 权限层级

平台的权限按照资源层级从上到下分为五个级别：

| 级别 | 范围 | 管理入口 |
|------|------|----------|
| 平台 | 整个平台的所有资源 | 控制台 > 访问控制 |
| 集群 | 特定集群内的资源 | 集群详情 > 集群设置 > 集群成员 |
| 租户空间 | 特定租户空间内的资源 | 租户空间详情 > 租户空间设置 > 成员 |
| 节点组 | 特定节点组内的资源 | 节点组详情 > 节点组设置 > 成员 |
| 项目 | 特定项目内的资源 | 项目详情 > 成员管理 |

## 平台角色

| 角色 | 权限范围 |
|------|----------|
| platform-admin | 管理平台所有资源，包括所有集群、租户空间和用户 |
| platform-self-provisioner | 可以创建租户空间，并自动成为新建租户空间的管理员 |
| platform-regular | 普通用户，需要被邀请加入租户空间后才能访问资源 |

## 角色类型

| 类型 | 说明 |
|------|------|
| 系统角色 | 平台内置角色，权限不可修改 |
| 自定义角色 | 由管理员创建的角色，可以自定义权限配置 |

## 权限分配原则

### 最小权限

只给用户分配完成工作所需的最低权限。例如，只需要查看集群信息的用户，分配集群查看者角色即可，不需要分配集群管理员角色。

### 按需分配

根据用户的实际工作职责分配权限：

| 工作职责 | 建议角色 |
|----------|----------|
| 平台运维 | platform-admin |
| 租户空间管理 | platform-self-provisioner + 对应租户空间管理员角色 |
| 应用开发 | platform-regular + 对应项目的开发角色 |
| 查看监控 | platform-regular + 对应集群的查看者角色 |

## 常见问题

### 问题：用户看不到某个菜单项
**现象**：用户登录后看不到控制台的某些菜单
**原因**：用户没有对应功能的权限
**解决**：检查用户的角色分配，确认是否有对应功能的访问权限

### 问题：用户操作被拒绝
**现象**：用户执行某个操作时提示权限不足
**原因**：用户当前的角色不包含该操作的权限
**解决**：为用户分配包含该操作权限的角色
