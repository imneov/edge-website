# 租户空间和工作台权限

> **所需权限**: 租户空间管理员或平台管理员

## 概述

除了平台级别的用户和角色管理外，租户空间和工作台内也有各自的成员管理功能。管理员可以邀请用户加入租户空间或工作台，并分配对应级别的角色。

## 租户空间成员管理

> **导航路径**: 租户空间详情 > 租户空间设置 > 租户空间成员
> **访问地址**: `/boss/clusters/{集群ID}/workspaces/{租户空间ID}/members`

### 操作一：查看成员列表

在租户空间的成员管理页面，可以查看当前租户空间的所有成员：

| 列 | 说明 |
|------|------|
| 成员名称 | 用户的用户名 |
| 邮箱 | 用户的邮箱地址 |
| 角色 | 用户在该租户空间中的角色 |
| 状态 | 用户账户状态 |
| 加入时间 | 加入该租户空间的时间 |

### 操作二：邀请成员

1. 在成员列表页面，点击 **邀请成员** 按钮
2. 搜索并选择要邀请的用户
3. 为用户选择在该租户空间中的角色
4. 点击 **确认**

被邀请的用户可以在工作台中访问该租户空间的资源。

### 操作三：修改成员角色

1. 点击目标成员右侧的操作菜单
2. 选择 **修改角色**
3. 选择新的角色
4. 点击 **确认**

### 操作四：移除成员

1. 点击目标成员右侧的操作菜单
2. 选择 **移除**
3. 确认移除

被移除的用户将无法再访问该租户空间的资源。

## 集群成员管理

> **导航路径**: 集群详情 > 集群设置 > 集群成员
> **访问地址**: `/boss/clusters/{集群ID}/members`

集群成员管理的操作方式与租户空间成员管理类似，区别在于分配的是集群级别的角色。

## 节点组成员管理

> **导航路径**: 节点组详情 > 节点组设置 > 节点组成员
> **访问地址**: `/boss/clusters/{集群ID}/nodegroups/{节点组ID}/members`

节点组成员管理用于控制用户对特定节点组资源的访问权限。

## 权限生效范围

| 成员类型 | 可访问的资源 |
|----------|------------|
| 租户空间成员 | 该租户空间下的所有项目和资源 |
| 集群成员 | 该集群下的所有资源 |
| 节点组成员 | 该节点组下的所有资源 |

## 常见问题

### 问题：邀请成员时找不到用户
**现象**：在邀请成员时搜索不到目标用户
**原因**：该用户尚未在平台中创建
**解决**：先在访问控制 > 用户管理中创建该用户，再进行邀请

### 问题：成员无法看到租户空间内的资源
**现象**：成员已加入租户空间，但在工作台看不到相关资源
**原因**：成员的角色权限不足，或成员未选择正确的工作台
**解决**：
1. 确认成员在工作台顶部选择了正确的租户空间
2. 检查成员的角色是否有查看权限
