# IAM（身份和访问管理）概述

## 什么是IAM？

IAM（Identity and Access Management，身份和访问管理）是边缘智能管理平台的核心安全模块，提供统一的用户身份管理、权限控制和访问审计功能。通过IAM，平台管理员可以精确控制不同用户对平台资源的访问权限，确保系统安全性和合规性。

## 核心功能

### 1. 用户管理
- **用户生命周期管理**：创建、查看、编辑、删除用户账号
- **用户信息维护**：管理用户的基本信息、联系方式、描述等
- **用户状态管理**：启用、禁用用户账号
- **密码管理**：重置用户密码，确保账号安全

### 2. 角色管理
- **分层角色体系**：支持平台级、集群级、工作空间级、项目级、节点组级角色
- **角色权限控制**：基于角色的访问控制（RBAC）
- **自定义角色**：创建和管理自定义角色模板
- **角色继承**：支持角色权限的继承和扩展

### 3. 权限管理
- **细粒度权限控制**：精确控制用户对不同资源的操作权限
- **角色模板**：预定义的角色权限模板，快速配置权限
- **权限测试**：权限测试工具，验证用户权限配置

### 4. 访问控制
- **多租户隔离**：基于租户的访问隔离
- **资源级授权**：支持不同层级资源的访问控制
- **操作审计**：记录用户操作日志，便于安全审计

## 访问IAM

### 进入方式
1. 登录边缘智能管理平台
2. 点击顶部导航栏中的"控制台"
3. 在左侧菜单中选择"访问控制"
4. 选择相应的管理功能（用户管理、角色管理等）

### 访问权限
只有具有管理员权限的用户才能访问IAM管理功能：
- **平台管理员**：拥有所有IAM管理权限
- **用户管理员**：拥有用户管理权限
- **租户空间管理员**：拥有其租户空间内的用户和角色管理权限

## IAM架构

### 分层权限模型

```
平台级（Platform）
├── 集群级（Cluster）
│   ├── 工作空间级（Workspace）
│   │   └── 项目级（Namespace）
└── 节点组级（NodeGroup）
```

### 权限继承关系
- 上层权限自动包含下层权限
- 同层权限相互独立，不相互影响
- 用户可以拥有多个层级的角色

## 核心概念

### 用户（User）
- 平台的访问主体，具有唯一的用户名
- 每个用户可以拥有一个或多个角色
- 用户信息包括基本资料、联系方式、状态等

### 角色（Role）
- 定义了一组权限的集合
- 角色分为系统内置角色和自定义角色
- 角色具有特定的作用范围（Scope）

### 权限（Permission）
- 定义了对特定资源的操作权限
- 包括读取、创建、更新、删除等操作
- 权限通过角色分配给用户

### 作用范围（Scope）
- 定义角色的有效范围
- 支持平台、集群、工作空间、项目、节点组等层级
- 不同层级的角色只能在其有效范围内生效

## 安全特性

### 1. 密码策略
- 强制密码复杂度要求
- 支持密码重置功能
- 密码传输加密保护

### 2. 会话管理
- 基于Token的身份验证
- 会话超时自动登出
- 支持多设备登录管理

### 3. 审计日志
- 记录所有用户操作
- 记录权限变更历史
- 支持日志查询和导出

### 4. 访问控制
- 基于角色的权限控制
- 资源级访问隔离
- 操作权限验证

## 最佳实践

### 用户管理
1. **最小权限原则**：只给用户分配其工作必需的最小权限
2. **定期审计**：定期检查用户权限，及时回收不需要的权限
3. **账号清理**：及时删除离职或转岗用户的账号

### 角色管理
1. **角色设计**：按照工作职能设计角色，避免过度细化
2. **权限组合**：合理组合权限，避免权限过于分散
3. **定期审查**：定期检查角色配置，确保权限设置合理

### 安全建议
1. **强密码策略**：启用密码复杂度要求，定期更换密码
2. **多因素认证**：在重要操作时启用多因素认证
3. **访问监控**：监控异常访问行为，及时发现安全威胁
