# 概念介绍

> **所需权限**: 平台管理员

## 核心概念

### 用户

用户是平台的登录账户。每个用户有唯一的用户名、邮箱和密码，可以登录平台并执行被授权的操作。

### 角色

角色定义了一组权限。将角色分配给用户后，用户就拥有了该角色所包含的所有权限。

### 角色范围

角色按作用范围分为以下几个级别：

| 范围 | 说明 | 示例 |
|------|------|------|
| platform | 平台级别，控制整个平台的访问权限 | 平台管理员、普通用户 |
| cluster | 集群级别，控制特定集群的访问权限 | 集群管理员、集群查看者 |
| workspace | 租户空间级别，控制特定租户空间的访问权限 | 租户空间管理员、租户空间成员 |
| nodegroup | 节点组级别，控制特定节点组的访问权限 | 节点组管理员 |
| namespace | 项目级别，控制特定项目的访问权限 | 项目管理员、项目查看者 |

### 权限继承

高级别的角色可以继承低级别的权限。例如，平台管理员自动拥有所有集群、租户空间和项目的管理权限。

## 内置角色

平台预置了以下系统角色：

| 角色 | 说明 |
|------|------|
| platform-admin | 平台管理员，管理平台上的所有资源 |
| platform-regular | 普通用户，被邀请加入租户空间之前无法访问任何资源 |
| platform-self-provisioner | 可以创建租户空间并成为所创建租户空间的管理员 |

## 权限分配方式

| 位置 | 操作 | 说明 |
|------|------|------|
| 访问控制 > 用户管理 | 创建用户时选择平台角色 | 分配平台级别权限 |
| 访问控制 > 角色管理 > 授权用户 | 为角色分配用户 | 批量分配权限 |
| 租户空间 > 成员管理 | 邀请成员并分配角色 | 分配租户空间级别权限 |
| 集群 > 成员管理 | 邀请成员并分配角色 | 分配集群级别权限 |
