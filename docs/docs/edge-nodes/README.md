# 边缘节点管理

> **导航路径**: 集群管理 / 租户空间 / 节点组 > 边缘节点
> **所需权限**: 平台管理员 或 集群管理员

边缘节点是部署在边缘侧的计算节点，通过边缘运行时（KubeEdge 或 OpenYurt）与云端集群保持连接，实现云边协同管理。

平台支持在创建集群时预置边缘运行时，也支持在集群创建后手动配置，并提供三种入口将边缘节点快速接入集群。

## 本节内容

| 文档 | 说明 |
|------|------|
| [配置集群边缘运行时](./cluster-runtime-setup) | 创建集群时选择 KubeEdge / OpenYurt，或对已有集群手动配置运行时 |
| [添加边缘节点](./add-edge-node) | 通过集群、节点组或租户空间三种入口将边缘节点接入平台 |

## 边缘运行时对比

| 特性 | KubeEdge | OpenYurt |
|------|----------|----------|
| 来源 | CNCF 边缘计算项目 | 阿里巴巴开源 |
| 核心能力 | 设备管理、边缘智能 | 节点自治、流量闭环 |
| 推荐场景 | IoT 设备接入、边缘 AI 推理 | 云边协同、弱网环境 |
| 平台标识 | `kubeedge` | `openyurt`（推荐） |

## 快速开始

1. **创建集群时选择运行时** → 参见 [配置集群边缘运行时](./cluster-runtime-setup)
2. **将边缘节点加入集群** → 参见 [添加边缘节点](./add-edge-node)
