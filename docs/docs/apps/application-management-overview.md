# 应用管理模块概述

## 简介

EdgePlatform 应用管理模块提供了一个企业级的应用商店和应用部署管理平台，支持应用的集中管理、分发和部署。用户可以通过应用商店浏览和部署应用，也可以通过应用管理控制台进行更高级的应用生命周期管理。

## 主要功能

### 1. 应用商店 (Application Store)
- **应用浏览**: 展示平台所有可用应用，支持按分类筛选
- **应用搜索**: 按应用名称快速查找目标应用
- **应用详情**: 查看应用的详细介绍、版本信息、使用说明等
- **一键部署**: 直接从应用商店部署应用到指定项目

### 2. 应用管理 (Application Management)
- **应用仓库管理**: 管理Helm Chart仓库和应用来源
- **应用审核**: 审核和上架用户提交的应用模板
- **版本管理**: 管理应用的多个版本，支持版本升级和回滚
- **应用部署**: 在工作空间和项目中部署应用实例

### 3. 应用实例管理
- **实例监控**: 查看应用实例的运行状态和资源使用情况
- **扩缩容**: 调整应用实例的副本数量
- **配置更新**: 更新应用的配置参数和环境变量
- **实例管理**: 启动、停止、重启应用实例

## 架构设计

### 应用类型

EdgePlatform 支持两种应用类型：

#### 1. 原生应用 (Native Applications)
- 基于Kubernetes工作负载（Deployment、StatefulSet等）
- 通过平台定义的工作负载配置进行部署
- 支持完整的应用生命周期管理

#### 2. Helm应用 (Helm Applications)
- 基于Helm Chart打包的应用
- 使用Helm进行部署和管理
- 支持复杂的微服务应用栈

### 应用分发流程

```
应用开发者 → 应用仓库 → 平台审核 → 应用商店 → 用户部署
```

1. **应用开发**: 开发者创建应用并上传到应用仓库
2. **仓库同步**: 平台从应用仓库同步应用信息
3. **应用审核**: 平台管理员审核应用质量和安全性
4. **上架发布**: 审核通过后发布到应用商店
5. **用户部署**: 用户从应用商店选择应用部署到项目

## 核心概念

### 应用 (Application)
应用的完整定义，包含应用的基本信息、描述、分类等元数据。

### 应用版本 (Application Version)
应用的特定版本，包含该版本的工作负载配置、Helm Chart等信息。

### 应用类别 (Application Category)
应用的分类标签，用于在应用商店中组织和筛选应用。

### 应用实例 (Application Instance)
部署在具体项目（namespace）中的应用运行实例。

### 应用仓库 (Application Repository)
存储Helm Chart的仓库地址，平台从仓库同步应用信息。

## 用户角色

### 平台管理员
- 管理应用仓库和分类
- 审核和上架应用
- 监控应用部署状态

### 工作空间管理员
- 在工作空间中部署应用
- 管理工作空间内的应用实例

### 开发者
- 提交应用到应用商店
- 管理自己的应用版本

### 普通用户
- 浏览应用商店
- 部署应用到有权限的项目

## 技术特性

### 1. 声明式管理
- 应用配置采用声明式YAML格式
- 支持版本控制和配置复用
- 便于自动化和持续集成

### 2. 多租户支持
- 基于工作空间和项目的多租户隔离
- 细粒度的权限控制
- 资源配额管理

### 3. 应用生命周期管理
- 支持应用的完整生命周期管理
- 版本升级和回滚
- 健康检查和自愈能力

### 4. 可观测性
- 应用状态监控
- 资源使用统计
- 日志和事件追踪

## 快速开始

### 浏览应用商店
1. 登录EdgePlatform控制台
2. 点击顶部导航栏的"应用商店"
3. 浏览或搜索目标应用

### 部署应用
1. 在应用商店中选择应用
2. 查看应用详情和版本信息
3. 点击"部署"按钮
4. 选择目标工作空间和项目
5. 配置应用参数
6. 完成部署

### 管理应用实例
1. 进入目标项目
2. 在左侧导航栏选择"应用"
3. 查看和管理应用实例
4. 进行扩缩容、配置更新等操作
