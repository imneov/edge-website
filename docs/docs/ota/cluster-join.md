# 集群纳管

> **导航路径**: 控制台 > OTA > 集群纳管
> **访问地址**: `/boss/ota/cluster-join`
> **所需权限**: 平台管理员

## 功能说明

集群纳管（Cluster Join）功能用于将已接入 OTA 系统的边缘节点纳入指定 Kubernetes 集群，并在节点上自动安装配置好容器运行时（containerd 或 Docker）及边缘框架组件。

每次纳管操作对应一个 **ClusterJoinTask**，底层通过 OTA `playbook` 任务类型（`playbookRef: cluster-join`）实现，支持同时对多个节点并行执行。

## 页面概览

集群纳管任务列表展示以下信息：

| 列 | 说明 |
|------|------|
| 任务名称 | 任务标识（可点击查看执行详情） |
| 状态 | 任务当前状态（等待中 / 运行中 / 已完成 / 失败） |
| 执行进度 | 成功节点数 / 总节点数，附带进度条 |
| 目标节点 | 参与纳管的节点列表 |
| 运行时 | 所使用的容器运行时及版本号 |
| 创建时间 | 任务创建时间 |
| 耗时 | 任务完成耗时 |

### 任务状态说明

| 状态 | 含义 |
|------|------|
| 等待中（Pending） | 任务已创建，等待调度 |
| 运行中（Running） | 节点正在执行安装步骤 |
| 已完成（Completed） | 所有节点均成功完成 |
| 失败（Failed） | 至少一个节点执行失败 |

## 操作指南

### 操作一：发起集群纳管

**前置条件**

- 目标节点已通过「节点接入」功能注册并处于在线状态
- 目标集群已在 Cluster 管理中创建，并已配置边缘框架（`cluster.theriseunion.io/edge-runtime` 注解）

**操作步骤**

1. 在集群纳管页面，点击右上角 **纳管节点** 按钮，打开纳管向导

2. **选择节点**（第 1 步）

   从节点列表中勾选需要纳管的边缘节点（可多选）

3. **选择集群**（第 2 步）

   选择目标 Kubernetes 集群。列表显示集群名称及其已配置的边缘框架类型（如 OpenYurt、KubeEdge）。

   > 仅显示已配置边缘框架注解的集群。若目标集群不在列表中，请先在集群管理中为其配置边缘运行时。

4. **配置运行时**（第 3 步）

   选择容器运行时及版本：

   | 运行时 | 支持版本 |
   |--------|----------|
   | containerd | 1.7.x、2.x |
   | Docker | 24.x、25.x、26.x、27.x |

5. **确认执行**（第 4 步）

   核对以下信息后点击 **开始纳管**：
   - 目标节点列表
   - 目标集群及边缘框架
   - 容器运行时及版本

**操作结果**

系统创建一个 ClusterJoinTask，任务出现在列表中，状态为「运行中」。

---

### 操作二：查看执行详情

**操作步骤**

1. 在任务列表中，点击任务名称

2. 在弹出的进度面板中查看每个节点的执行步骤和日志

**操作结果**

可实时监控各节点的安装进度，排查失败原因。

---

### 操作三：删除任务

**操作步骤**

1. 在任务行右侧点击操作菜单（`⋯`）

2. 选择 **删除**

3. 在确认对话框中输入任务名称后点击 **确认删除**

**操作结果**

任务及其所有执行记录被删除，此操作不可恢复。

> 删除任务不会卸载已安装到节点上的组件。

## 常见问题

### 问题：集群列表为空，无法选择目标集群
**原因**：所有集群均未配置边缘框架注解
**解决**：进入集群管理页面，为目标集群添加 `cluster.theriseunion.io/edge-runtime` 注解（值为 `openyurt` 或 `kubeedge`）

### 问题：任务状态长时间停留在「运行中」
**原因**：节点网络不稳定、安装包下载超时或节点权限不足
**解决**：点击任务名称查看各节点的详细执行日志，根据错误信息排查

### 问题：部分节点失败，其余节点成功
**原因**：每个节点独立执行，互不影响
**解决**：查看失败节点的日志，修复问题后重新对失败节点发起纳管任务
