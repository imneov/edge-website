# 节点组管理

> **导航路径**: 集群 > 选择集群 > 节点组
> **访问地址**: `/boss/clusters/{集群ID}/nodegroups`
> **所需权限**: 平台管理员或集群管理员

## 功能说明

节点组用于将集群中的节点按逻辑分组管理。每个节点组关联一个项目，节点组内的节点共享资源监控和管理。通过节点组，您可以按业务或位置组织节点，查看各组的资源使用情况。

## 页面概览

![节点组列表](./images/cluster-nodegroups-list.png)

节点组列表包含以下信息：

| 列 | 说明 |
|------|------|
| 名称 | 节点组的名称和标识 |
| 节点数 | 节点组中包含的节点数量 |
| CPU 使用率 | 节点组的平均 CPU 使用率 |
| 内存使用率 | 节点组的平均内存使用率 |
| 磁盘使用率 | 节点组的平均磁盘使用率 |
| 已授权项目数量 | 关联的项目数量 |
| 创建时间 | 节点组的创建时间 |

## 操作指南

### 操作一：查看节点组列表

**操作步骤**

1. 进入目标集群，在侧边栏点击 **节点组**

2. 查看所有节点组的名称、节点数和资源使用率

3. 使用搜索框按名称查找节点组

**操作结果**

查看到集群中所有节点组及其资源使用概况。

### 操作二：创建节点组

**操作步骤**

1. 在节点组列表页面，点击 **创建节点组** 按钮

2. 填写节点组信息：

   | 字段 | 是否必填 | 说明 |
   |------|----------|------|
   | 节点组名称 | 是 | 节点组的显示名称 |
   | 描述 | 否 | 节点组的用途说明 |

3. 选择要加入节点组的节点

4. 点击 **确认** 创建节点组

**操作结果**

节点组创建成功，出现在节点组列表中。

### 操作三：查看节点组详情

**操作步骤**

1. 在节点组列表中，点击目标节点组的名称

2. 进入节点组详情页，查看以下信息：
   - 节点组内的节点列表和状态
   - 各节点的资源使用情况
   - 关联的项目信息

**操作结果**

了解节点组内各节点的运行状态和资源分布。

### 操作四：删除节点组

**操作步骤**

1. 在节点组列表中，点击目标节点组右侧的操作按钮

2. 选择 **删除**

3. 确认删除操作

**操作结果**

节点组被删除。删除节点组不会删除节点本身，只是解除分组关系。

## 常见问题

### 问题：节点组资源使用率显示为 "-"
**现象**：某些节点组的 CPU、内存、磁盘使用率显示为 "-"
**原因**：节点组内没有在线节点，或节点尚未上报监控数据
**解决**：检查节点组内的节点是否在线并正常运行

### 问题：创建节点组时没有可选节点
**现象**：创建节点组时节点列表为空
**原因**：集群中没有可用节点，或所有节点已分配到其他节点组
**解决**：确认集群中有可用节点；如需要，先从其他节点组移除节点
