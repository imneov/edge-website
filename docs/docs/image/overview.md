# 概念介绍

> **所需权限**: 平台管理员

## 核心概念

镜像管理模块包含两个核心概念：

### 镜像源

镜像源是**镜像仓库的访问凭证**，包含仓库地址、用户名和密码。每个镜像源对应一个外部镜像仓库（如 Harbor、Docker Hub）的登录信息。

**类比**：镜像源就像银行账户的登录凭证，告诉系统如何连接到镜像仓库。

### 镜像仓库

镜像仓库是**需要从镜像源同步的具体镜像**。创建镜像仓库时，需要指定使用哪个镜像源以及要同步哪个镜像路径。

**类比**：镜像仓库就像要从银行取出的具体资产，需要先有账户（镜像源）才能操作。

## 两者的关系

| 概念 | 作用 | 举例 |
|------|------|------|
| 镜像源 | 存储仓库的登录凭证 | Harbor 服务器的用户名和密码 |
| 镜像仓库 | 指定要同步的具体镜像 | `edge/apiserver` 镜像 |

一个镜像源可以被多个镜像仓库引用。例如，同一个 Harbor 凭证可以用于同步 `edge/apiserver`、`edge/demo` 等多个镜像。

## 使用场景

| 场景 | 使用方式 |
|------|----------|
| 部署边缘应用 | 先添加镜像源，再添加应用所需的镜像 |
| 离线环境部署 | 提前同步镜像到边缘集群 |
| 多项目共享镜像 | 在不同项目中配置各自的镜像源和镜像 |
