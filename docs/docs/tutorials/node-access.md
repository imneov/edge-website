# 教程1：节点接入

将一台 Linux 机器接入平台，完成后节点出现在工作台的节点资源列表中。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| 已有租户空间 | 平台已创建租户空间并分配给你 |
| 网络可达 | 边缘节点能访问平台地址（默认端口 30080） |
| Root 权限 | 目标机器具有 root 权限 |

---

## 第一步：进入节点资源

登录平台后，点击顶部导航「**边缘工作台**」，进入租户空间列表。

![工作台首页](/img/tutorials/workbench-home.png)

点击目标租户空间的「**查看详情**」，进入工作台。

![租户空间列表](/img/tutorials/express-workspace.png)

在左侧导航展开「**边缘资源**」，点击「**节点资源**」。

![节点资源页面](/img/tutorials/node-resources.png)

---

## 第二步：生成安装命令

点击右上角「**+ 加入节点**」。

![加入节点对话框](/img/tutorials/add-node-dialog.png)

在弹出的对话框中：

1. **容器运行时**：选择 `Containerd`（推荐）
2. **名称**：填写节点名称，如 `edge-node-01`
3. 点击「**验证**」按钮

验证通过后，对话框展示「**边缘节点配置命令**」。

![安装命令](/img/tutorials/node-install-command.png)

点击右侧复制图标，将命令复制到剪贴板。

---

## 第三步：执行安装

SSH 登录到目标边缘节点，粘贴并执行复制的命令：

```bash
# 粘贴对话框中复制的命令，例如：
test -n "188.239.62.17" && ...
curl -sS http://bin-downloader.rise.io:30080/scripts/install_kubeedge_env.sh | ...
```

命令执行完毕后，回到平台节点资源页面，点击「**刷新**」。节点状态变为「**运行中**」即表示接入成功。

---

## 预期结果

- 节点出现在节点资源列表
- 状态显示「运行中」
- CPU、内存信息正常显示

:::tip
如果节点状态长时间显示「未就绪」，请检查边缘节点到平台的网络连通性（需能访问平台 30080 端口）。
:::
