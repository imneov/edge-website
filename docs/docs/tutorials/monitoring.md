# 教程4：监控展示

通过监控中心，一屏掌握集群拓扑和所有节点的运行状态。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| 已有集群 | 平台已配置至少一个集群 |
| 节点已接入 | 有运行中的节点才能看到监控数据 |

---

## 第一步：打开监控中心

点击顶部导航「**监控中心**」，进入拓扑视图。

![监控中心拓扑视图](/img/tutorials/monitoring-center.png)

拓扑图以树状结构展示：
- **顶层**：Host 集群（平台控制节点）
- **第二层**：子集群（虚拟集群 ke003、ke002、oy001 等）
- **第三层**：各租户空间的节点组
- **底层**：具体边缘节点

每个节点卡片显示实时 CPU 使用率、内存用量、节点数量。

---

## 第二步：查看集群节点列表

点击某个子集群卡片（如「ke003」），进入该集群的节点监控列表。

![集群节点监控](/img/tutorials/cluster-metrics.png)

节点列表分为两组：
- **云端节点**：运行在云侧的主节点
- **边缘节点**：部署在边缘的工作节点

---

## 第三步：查看节点详情

点击任意节点卡片，查看该节点的详细监控数据：

| 指标 | 说明 |
|------|------|
| CPU 用量 | 实时 CPU 使用百分比及趋势图 |
| 内存用量 | 物理内存使用量 |
| 网络流量 | 入/出流量实时数据 |
| Pod 数量 | 当前运行的容器组数量 |

---

## 预期结果

- 拓扑图完整展示集群层级关系
- 节点卡片显示实时 CPU、内存数据
- 点击节点可查看详细指标和历史趋势

:::tip
拓扑图支持缩放：使用左下角的 `+` / `-` 按钮，或滚动鼠标滚轮调整视图大小。点击右下角的全屏按钮可以全屏查看。
:::
