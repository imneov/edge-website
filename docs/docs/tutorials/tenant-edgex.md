# 教程5：创建租户空间与启用 EdgeX

创建新的租户工作空间，并开启 EdgeX Foundry IoT 平台服务，用于边缘设备数据采集和协议转换。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| 管理员权限 | 创建租户需要平台管理员或租户管理员权限 |
| 已有集群 | 至少有一个可用集群用于承载租户空间 |

---

## 第一步：创建租户空间

进入「**租户空间**」管理页面（`/boss/workspaces`），点击「**+ 创建租户空间**」。

![创建租户空间对话框](/img/tutorials/create-tenant-dialog.png)

填写以下信息：

| 字段 | 说明 |
|------|------|
| **工作空间名称** | 小写字母、数字、`-` 或 `_`，最长 63 字符 |
| **别名** | 显示名称，可包含中文 |
| **所属集群** | 选择承载该租户的集群 |
| **描述** | 可选，说明租户用途 |

点击「**创建**」，租户空间创建成功后出现在列表中。

---

## 第二步：进入工作台

在租户列表点击「**查看详情**」，进入工作台概览页。

![工作台概览（含 EdgeX 按钮）](/img/tutorials/enable-edgex.png)

右侧信息面板显示该租户空间的基本信息，其中「**IoT 平台服务**」默认显示「EdgeX 未启用」。

---

## 第三步：启用 EdgeX

点击右上角「**启用 EdgeX**」按钮。

系统会在该租户空间的集群中部署 EdgeX Foundry 服务，包括：

- **Core Services**：核心数据、命令、元数据服务
- **Device Services**：设备协议适配层（Modbus、OPC-UA 等）
- **Application Services**：数据导出和规则引擎

等待部署完成（约 2-5 分钟），刷新页面后，IoT 平台服务状态变为「**EdgeX 已启用**」。

---

## 预期结果

- 租户空间创建成功，出现在工作台列表
- EdgeX 服务成功启动，状态显示「已启用」
- 左侧导航「**设备管理**」和「**设备模版**」功能可正常访问

:::info
EdgeX 启用后，可通过「边缘资源」→「设备管理」添加和管理 IoT 设备，支持 Modbus、BACnet、OPC-UA 等工业协议。
:::

:::tip
如果 EdgeX 部署失败，通常是集群资源不足。确认节点有足够的 CPU（≥2 核）和内存（≥2GB）可用。
:::
