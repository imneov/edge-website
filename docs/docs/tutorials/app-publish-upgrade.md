# 教程3：应用上架与升降级

将应用发布到应用商店，让其他租户也能使用；并通过版本控制实现应用升级和回滚。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| 已创建应用模板 | 参考[教程2](./app-deployment)，应用状态为「草稿」 |
| 管理员账号 | 审核步骤需要管理员权限 |

---

## 第一步：提交审核

进入租户空间 → 「**应用列表**」，点击应用名称进入详情页。

![应用模板详情](/img/tutorials/submit-review.png)

在版本区域，点击右侧「**⋯**」操作按钮，选择「**提交审核**」。

- 填写审核说明（可选）
- 点击「**确认**」

提交后，版本的审核状态变为「**待审核**」。

---

## 第二步：管理员审核

切换到管理员视角，点击右上角「**控制台**」，进入应用商店管理。

![应用商店管理](/img/tutorials/app-store-management.png)

左侧点击「**应用审核**」：

![应用审核列表](/img/tutorials/app-review.png)

找到待审核的应用版本，点击「**详情**」，完成审核（通过 ✅ 或拒绝 ❌）。

审核通过后，回到「**应用**」列表，点击应用名称，在详情页点击「**上架**」。

---

## 第三步：其他租户部署公共应用

切换到另一个租户空间，进入「**推理服务**」，点击「**+ 创建服务**」。

点击「**公共应用**」标签，可以看到刚上架的应用：

![创建服务选择公共应用](/img/tutorials/create-workload.png)

选择应用，配置部署参数（注意修改主机端口避免冲突），点击「**创建**」。

---

## 第四步：应用升降级

**发布新版本：**

在应用模板详情页，点击「**+ 应用版本**」，填写新版本号（如 `2.0.0`），配置容器镜像，创建新版本。

**切换版本：**

进入「**推理服务**」，找到已部署的应用，在操作列点击「**⋯**」→「**版本控制**」，选择目标版本（升级或回滚），点击「**切换**」。

---

## 预期结果

- 应用成功上架，可在应用商店中查到
- 其他租户可直接使用公共应用部署
- 版本切换后，工作负载自动更新至目标版本

:::tip
版本切换不会删除旧版本数据，随时可以回滚。
:::
