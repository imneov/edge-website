# 教程2：应用部署

从镜像接入到应用运行，完成一次完整的边缘应用部署。

---

## 前置条件

| 条件 | 说明 |
|------|------|
| 已有租户空间 | 平台已分配租户空间 |
| 已接入节点 | 至少有一个运行中的边缘节点（参考[教程1](./node-access)） |
| 镜像仓库地址 | 已有可用的 Docker 镜像仓库（Docker Hub、Harbor 等） |

---

## 第一步：接入镜像仓库

进入租户空间，左侧导航展开「**边缘应用**」，点击「**模型列表**」。

![模型列表页面](/img/tutorials/image-registry.png)

点击「**+ 添加模型**」，在弹出的「选择镜像源」面板中，可以看到已配置的镜像仓库列表。

![选择镜像源](/img/tutorials/add-image-source.png)

:::info
如果提示「暂无镜像源」，请先联系管理员在控制台配置镜像仓库。
:::

---

## 第二步：创建应用模板

左侧导航点击「**应用列表**」。

![应用列表](/img/tutorials/app-templates.png)

点击「**+ 新建应用**」，按向导填写应用信息：

![创建应用模板](/img/tutorials/create-app-template.png)

| 步骤 | 填写内容 |
|------|---------|
| 基本信息 | 应用名称、版本（如 `1.0.0`）、别名 |
| 容器组设置 | 选择镜像、配置容器端口 |
| 容器网络设置 | 配置主机端口映射 |
| 标签及注解 | 可选，按需填写 |

逐步点击「**下一步**」，最后点击「**创建**」。

---

## 第三步：部署应用

左侧导航点击「**推理服务**」。

![推理服务（工作负载）](/img/tutorials/workloads.png)

点击「**+ 创建服务**」，选择刚创建的应用：

![创建推理服务](/img/tutorials/create-workload.png)

对话框显示两个标签页：
- **租户应用**：本租户创建的应用
- **公共应用**：应用商店中上架的应用

选择目标应用，点击进入配置：

1. 查看容器组配置信息
2. 选择部署到的**边缘节点**
3. 点击「**创建**」

---

## 预期结果

- 工作负载列表中出现新部署的应用
- 状态变为「运行中」
- 可通过配置的主机端口访问应用

:::tip
部署后状态显示「等待中」属正常，镜像拉取需要一定时间，等待1-3分钟后刷新即可。
:::
