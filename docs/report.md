## 欢迎使用边缘智能管理平台

**边缘智能管理��台**是一款面向云原生应用的智能边缘计算平台，支持 Kubernetes 分布式边缘基础设施的无缝管理。

### 平台简介

边缘智能管理平台将 Kubernetes 能力延伸至边缘，提供：

- **云原生边缘管理**：使用标准 Kubernetes API 管理边缘节点和工作负载

- **统一可观测性**：跨云端和边缘的集中监控和日志
- **安全与合规**：内置安全策略和合规控制

### 核心功能

#### 多集群管理
- 单一控制平面管理多个边缘集群
- 跨集群工作负载编排
- 统一资源管理和监控

#### 边缘原生能力
- 断网场景下的节点自治
- 边缘节点自动注册和发现
- 本地数据处理和聚合

#### 多租户支持
- 基于工作空间的资源隔离
- 细粒度角色权限控制（RBAC）
- 资源配额管理

#### 企业级特性
- 审计日志和合规性
- 高可用和灾难恢复
- 完善的 API 支持

## 产品概述

### 什么是边缘智能管理平台？

边缘智能管理平台是一款基于 Kubernetes 构建的云原生边缘计算平台，旨在将云能力扩展至边缘位置，同时保持操作的一致性和简洁性。

### 核心能力

#### 1. 边缘集群管理

像管理单个 Kubernetes 集群一样轻松管理分布式边缘基础设施：

- **集中控制**：所有边缘集群的单一控制平面
- **统一 API**：云边统一的 标准 Kubernetes API
- **集群生命周期**：自动化的集群配置和升级
- **健康监控**：实时集群健康状态跟踪

#### 2. 工作负载编排

在边缘位置智能地部署和管理应用：

- **自动调度**：基于资源和策略的智能工作负载调度
- **边缘自治**：云断连期间继续运行
- **滚动更新**：零停机应用更新
- **资源优化**：跨集群高效资源利用

#### 3. 多租户与访问控制

企业级安全和隔离：

- **工作空间隔离**：团队和项目的逻辑隔离
- **RBAC**：细粒度的基于角色的访问控制
- **网络策略**：安全的网络通信
- **审计日志**：完整的合规审计追踪

#### 4. 可观测性

全面的监控和故障排查：

- **统一监控**：基于 Prometheus 的指标采集
- **集中日志**：所有边缘位置的日志聚合
- **告警管理**：可配置的关键事件告警
- **链路追踪**：端到端请求追踪

### 技术优势

#### 云原生架构

基于成熟的云原生技术构建：
- **Kubernetes**：行业标准容器编排
- **Operators**：Kubernetes 原生生命周期管理
- **CRDs**：可扩展的资源定义
- **Helm**：标准化应用打包

#### 生产就绪

为企业部署设计：
- **高可用性**：多副本组件和故障转移
- **可扩展性**：支持数千个边缘节点
- **安全性**：TLS 加密、证书管理、密钥管理
- **合规性**：审计日志、数据驻留控制

#### 开发者友好

为生产力而构建：
- **GitOps 就绪**：原生集成 ArgoCD 和 Flux
- **REST API**：全面的 RESTful API
- **CLI 工具**：自动化命令行界面
- **Web 控制台**：直观的 Web 管理界面

### 应用场景

边缘智能管理平台在以下场景中表现出色：

1. **低延迟处理**：边缘位置的实时数据处理
2. **带宽优化**：降低数据传输成本
3. **离线运行**：网络中断期间继续运行
4. **数据主权**：满足数据驻留合规要求

### 组件详解

#### 云控制平面

##### edge-apiserver

中央 API 网关提供以下功能：

- **身份认证**：OAuth2/OIDC 集成、令牌验证
- **权限授权**：RBAC、策略评估
- **API 网关**：所有资源的 RESTful API 端点
- **多租户**：工作空间和命名空间隔离
- **审计日志**：完整的请求/响应日���

**核心特性：**
- 基于 Kubernetes API 机制构建
- 通过自定义资源定义（CRD）扩展
- 多副本高可用部署
- 所有通信使用 TLS 加密

##### edge-controller

负责编排的协调引擎：

- **集群生命周期**：注册、健康检查、下线
- **资源调和**：期望状态与实际状态对比
- **策略执行**：放置策略、资源配额
- **工作负载分发**：跨集群智能调度
- **配置管理**：集中化配置分发

##### edge-console

基于 Web 的管理界面提供：

- **仪表板**：集群、节点、工作负载概览
- **资源管理**：所有资源的 CRUD 操作
- **用户管理**：用户、角色和权限管理
- **监控集成**：指标仪表板和图表
- **审计查看器**：搜索和过滤审计日志

#### 监控服务

可观测性基础设施包括：

- **Prometheus**：时序指标数据库
- **Alertmanager**：告警路由和通知
- **Grafana**：指标可视化（可选）
- **ReverseProxy**：多集群指标聚合

### 边缘组件

#### Kubernetes 集群

每个边缘位置运行标准 Kubernetes 集群：

- **控制平面**：API 服务器、控制器管理器、调度器
- **工作节点**：kubelet、容器运行时（containerd/docker）
- **网络**：CNI 插件（Calico、Flannel 等）
- **存储**：持久卷的 CSI 驱动

#### Edge Agent

运行在边缘集群的本地代理：

- **注册**：自动向控制平面注册集群
- **心跳**：定期上报健康状态
- **资源同步**：双向资源同步
- **本地缓存**：缓存频繁访问的数据
- **指标导出**：向控制平面转发指标

### 数据流

#### 1. 用户请求流程

```
用户 → Console/CLI → edge-apiserver → 身份认证 → 权限授权 → CRD Controller → Kubernetes API
```

#### 2. 集群注册流程

```
边缘集群 → Edge Agent → edge-apiserver → edge-controller → 集群 CRD 创建 → 状态更新
```

#### 3. 工作负载部署流程

```
用户 → edge-apiserver → 工作负载 CRD → edge-controller → 目标集群选择 → Edge Agent → Kubernetes 应用
```

#### 4. 监控流程

```
边缘集群指标 → Prometheus → ReverseProxy → 监控服务 → edge-console 仪表板
```

## 应用场景

边缘智能管理平台旨在解决边缘计算场景中的实际挑战。以下是边缘平台表现出色的常见应用场景。

### 1. IoT 数据处理

#### 场景
制造工厂中有数千个 IoT 传感器生成实时数据，需要立即处理。

#### 挑战
- 海量数据占满云带宽
- 延迟敏感的决策（< 100ms）
- 网络连接不稳定
- 数据主权合规要求

#### 边缘平台解决方案
- **本地处理**：在边缘位置处理传感器数据
- **数据聚合**：在上传前减少数据量
- **边缘自治**：网络中断期间继续运行
- **实时分析**：毫秒级决策

#### 收益
- 云带宽成本降低 90%
- 处理延迟低于 10ms
- 网络问题下仍保持 99.9% 可用性
- 符合本地数据合规要求

### 2. 内容分发与 CDN

#### 场景
媒体流服务为数百万不同地理区域的用户提供服务。

#### 边缘平台解决方案
- **边缘缓存**：在边缘位置部署缓存服务
- **地理分布**：自动将用户路由到最近的边缘
- **智能预取**：预测并缓存热门内容
- **自适应流**：根据本地网络调整质量

#### 收益
- 源服务器负载降低 70%
- 流式传输延迟降低 50%
- 带宽成本降低 60%

### 3. 零售与销售点

#### 场景
零售连锁店拥有数百家门店，运行 POS 系统、库存管理和客户分析。

#### 边缘平台解决方案
- **门店级集群**：每个门店作为一个边缘集群
- **离线运营**：POS 无需互联网即可工作
- **数据同步**：连接恢复后自动同步
- **集中可见性**：从总部监控所有门店

#### 收益
- 门店运营零停机
- 实时库存可见性
- 符合 GDPR/CCPA 标准

### 4. 智慧城市基础设施

#### 场景
全市范围部署智能交通灯、停车传感器和环境监测设备。

#### 边缘平台解决方案
- **区域级集群**：每个城市区域部署边缘集群
- **多租户**：为不同部门提供独立访问
- **实时控制**：亚秒级交通灯调整
- **统一监控**：全市范围仪表板

#### 收益
- 交通拥堵减少 30%
- 公共安全响应时间缩短

### 5. 电信与 5G

#### 场景
电信运营商部署具有边缘计算能力的 5G 网络，为企业客户提供服务。

#### 边缘平台解决方案
- **MEC 集成**：多接入边缘计算支持
- **网络切片**：每个租户的隔离虚拟网络
- **动态扩缩容**：基于需求自动扩展
- **服务编排**：自动化服务部署

#### 收益
- 5G 应用亚 1ms 延迟
- 支持数千个边缘站点
- 99.999% 服务可用性

### 6. 医疗与医学影像

#### 场景
医院网络拥有医学影像设备，需要实时处理和 HIPAA 合规。

#### 边缘平台解决方案
- **本地处理**：在本地处理影像
- **AI/ML 推理**：在边缘部署 ML 模型
- **数据驻留**：敏感数据保留在本地
- **安全集成**：TLS 加密连接

#### 收益
- 立即获得影像处理结果
- 100% HIPAA 合规
- 敏感数据不离开医院

## 前置要求

在开始使用边缘平台之前，请确保满足以下前置条件。

### 系统要求

#### 客户端要求

**客户端系统配置**：
- **操作系统**: Windows 10+, macOS 10.15+, Linux (Ubuntu 18.04+, CentOS 7+)
- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **屏幕分辨率**: 最低 1366×768, 推荐 1920×1080
- **网络**: 稳定的互联网连接, 推荐带宽 ≥ 10Mbps

#### 服务器要求

边缘平台运行在 Kubernetes 集群之上，需要满足以下要求：

**最小配置（POC 环境）**：
- **节点数量**: 1 个控制平面节点 + 2 个工作节点
- **CPU**: 每节点 4 核
- **内存**: 每节点 8GB
- **存储**: 每节点 100GB SSD
- **网络**: 节点间网络延迟 < 10ms

**生产环境配置**：
- **节点数量**: 3 个控制平面节点 + 5+ 个工作节点
- **CPU**: 每节点 8 核+
- **内存**: 每节点 16GB+
- **存储**: 每节点 200GB+ SSD
- **网络**: 高可用网络，节点间网络延迟 < 5ms

### 软件依赖

#### Kubernetes 集群

**支持的 Kubernetes 发行版**：

| 发行版 | 版本要求 | 推荐版本 |
|--------|----------|----------|
| **Kubernetes** | ≥ 1.24.0 | 1.28.x |
| **OpenYurt** | ≥ 1.4.0 | 1.6.x |
| **K3s** | ≥ 1.24.0 | 1.28.x |
| **RKE** | ≥ 1.4.0 | 1.5.x |

> 推荐使用 OpenYurt：边缘平台针对 OpenYurt 进行了深度优化，提供最佳的边缘计算体验。

#### 存储后端

边缘平台需要持久化存储支持：
- **Local Path Provisioner** (开发/测试环境)
- **NFS** (生产环境)
- **Ceph RBD** (生产环境)
- **云存储** (AWS EBS, Azure Disk, GCP PD)

#### 监控组件（可选）

如果需要使用监控功能，需要部署：
- **Prometheus** ≥ 2.40.0
- **Grafana** ≥ 9.0.0

### 访问权限

#### 管理员权限

安装边缘平台需要以下权限：
- Kubernetes 集群 **cluster-admin** 权限
- SSH 访问所有节点的权限
- 防火墙和网络配置权限

#### 用户权限

使用边缘平台的用户需要：
- 平台账号（由管理员创建）
- 分配的角色和权限
- 网络访问边缘平台 Console 的权限

**生产环境推荐配置**：
- 为 Console 配置独立域名（如 `console.edge-platform.io`）
- 为 API Server 配置独立域名（如 `api.edge-platform.io`）
- 使用有效的 TLS 证书（Let's Encrypt 或企业 CA）

## 创建第一个集群

本文档将指导管理员如何在边缘平台中添加和管理第一个 Kubernetes 集群。

### 前置条件

在开始之前，请确保：
- 已完成首次登录
- 具有**平台管理员**权限
- 已有运行中的 Kubernetes 集群
- 有集群的 kubeconfig 文件

> 支持 Kubernetes 1.24+ 版本，推荐使用 OpenYurt 1.6+ 以获得更好的边缘计算能力。

### 集群添加方式

边缘平台支持两种方式添加集群：

#### 方式 1: 导入现有集群（推荐）

适用于已有 Kubernetes 集群的场景。

**优点**：
- 快速接入
- 不改变集群配置
- 支持多集群管理

#### 方式 2: 通过安装器创建集群

使用边缘平台提供的安装器创建新集群。

**优点**：
- 自动化部署
- 最佳实践配置
- 集成边缘计算组件

### 步骤 1: 准备 Kubeconfig

#### 获取 Kubeconfig 文件

从 Kubernetes 集群获取 kubeconfig 文件：

```bash
# 方式 1: 从主节点复制
scp root@<master-ip>:~/.kube/config ./kubeconfig

# 方式 2: 导出当前配置
kubectl config view --raw > ./kubeconfig

# 方式 3: 使用特定 context
kubectl config view --raw --context=<context-name> > ./kubeconfig
```

#### 验证 Kubeconfig

确保 kubeconfig 文件有效：

```bash
# 测试连接
kubectl --kubeconfig=./kubeconfig get nodes

# 检查版本
kubectl --kubeconfig=./kubeconfig version
```

**预期输出**：

```bash
$ kubectl --kubeconfig=./kubeconfig get nodes
NAME       STATUS   ROLES           AGE   VERSION
master-1   Ready    control-plane   10d   v1.28.0
worker-1   Ready    <none>          10d   v1.28.0
worker-2   Ready    <none>          10d   v1.28.0
```

> **安全提示**：Kubeconfig 包含集群访问凭证，请妥善保管。不要提交到版本控制系统，不要通过不安全的渠道传输。

### 步骤 2: 在 Console 中添加集群

#### 进入集群管理页面

1. 登录边缘平台 Console
2. 点击左侧菜单"平台管理" > "集群管理"
3. 点击右上角"添加集群"按钮

#### 填写基本信息

在"添加集群"对话框中填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| **集群名称*** | 集群的唯一标识 | `prod-cluster` |
| **显示名称** | 在 Console 中显示的名称 | `生产环境集群` |
| **集群描述** | 集群的用途描述 | `生产环境主集群，运行核心业务` |
| **集群类型** | 集群类型选择 | `Kubernetes` / `OpenYurt` |
| **环境标签** | 环境标识 | `生产` / `测试` / `开发` |

> 集群名称只能包含小写字母、数字和连字符（-），且必须以字母开头。

#### 上传 Kubeconfig

**方式 1: 文件上传**

1. 点击"上传 Kubeconfig"按钮
2. 选择准备好的 kubeconfig 文件
3. 系统会自动解析并验证

**方式 2: 直接粘贴**

1. 点击"直接输入"标签
2. 将 kubeconfig 内容粘贴到文本框
3. 点击"验证"按钮

#### 验证连接

点击"测试连接"按钮，系统会：
1. 验证 kubeconfig 格式
2. 测试集群连通性
3. 检查必要权限
4. 获取集群基本信息

#### 完成添加

确认所有信息无误后，点击"确定"按钮完成集群添加。

### 步骤 3: 验证集群状态

#### 查看集群列表

在"集群管理"页面，您可以看到刚添加的集群：

```
┌─────────────────────────────────────────────────────────────┐
│ 集群名称        │ 状态  │ 节点  │ Kubernetes 版本 │ 更新时间   │
├─────────────────────────────────────────────────────────────┤
```

#### 访问控制

   - **集群管理员** - 完全控制权限

#### 更新 Kubeconfig

如果集群凭证更新：

## 工作台概述

工作台是边缘智能管理平台的核心工作区域，每个工作台对应一个独立的边缘计算环境。您可以在工作台中管理边缘节点、部署应用、配置服务，并管理团队成员的访问权限。

### 页面概览

进入「智能边缘平台」后，您将看到所有可访问的工作台卡片列表。每张卡片显示工作台的名称、ID、运行状态、所属集群和管理员信息。

### 工作台侧边栏

进入某个工作台后，左侧显示功能导航菜单：

| 菜单分组 | 子菜单 | 说明 |
|----------|--------|------|
| 总览 | — | 查看工作台资源使用概览和基本信息 |
| 边缘资源 | 节点资源 | 管理工作台内的边缘节点 |
| 边缘资源 | 算力管理 | 查看和管理 GPU 资源 |
| 边缘资源 | 设备管理 | 管理 IoT 设备 |
| 边缘资源 | 设备模版 | 管理设备配置模版 |
| 边缘应用 | 工作负载 | 管理部署、有状态副本集等工作负载 |
| 边缘应用 | 应用模板 | 浏览和管理应用模板 |
| 边缘应用 | 镜像仓库 | 管理镜像源 |
| 边缘应用 | 算法服务 | 管理算法服务镜像 |
| 配置管理 | 配置字典 | 管理应用配置数据 |
| 配置管理 | 保密字典 | 管理敏感信息 |
| 租户设置 | 租户空间角色 | 查看和管理角色 |
| 租户设置 | 租户空间成员 | 邀请和管理成员 |

### 快速导航

- **选择和切换工作台** — 进入工作台、查看概览信息
- **边缘资源管理** — 管理节点和设备

## 监控平台

监控中心提供集群和节点的实时资源监控。通过拓扑视图可以直观了解所有集群、节点组和虚拟集群的运行状态和资源使用情况。

### 页面概览

监控中心首页以拓扑图形式展示整个平台的集群架构：

- **中心节点**：Host 集群，显示集群类型、边缘框架、告警状态、CPU/内存使用率、节点数和容器组数
- **节点组**：Host 集群下属的各节点组，显示名称、状态和节点数量
- **虚拟集群**：独立运行的虚拟集群，显示 CPU/内存使用率、节点数和容器组数
- **Member 集群**：加入平台的成员集群

### 入口说明

监控功能有两个入口：

| 入口 | 导航路径 | 功能 |
|------|----------|------|
| 监控中心 | 顶部导航栏 > 监控中心 | 查看拓扑全景、进入节点监控 |
| 集群监控告警 | 集群 > 选择集群 > 监控告警 | 查看集群指标、告警、趋势图 |

### 快速导航

| 文档 | 说明 |
|------|------|
| 集群节点监控 | 查看集群内所有节点的资源使用情况 |
| 集群监控指标 | 查看集群整体的 CPU、内存、磁盘和网络指标 |
| 告警管理 | 查看活跃告警和创建告警规则 |
| 趋势分析 | 查看 CPU 和内存的历史趋势图表 |

## OTA 管理概述

OTA（Over-the-Air）模块用于远程管理边缘节点，支持节点注册、状态监控、远程命令执行、文件读写和批量任务管理。通过 OTA 模块，管理员可以对大规模边缘节点进行统一运维。

### OTA 功能

OTA 模块包含两个主要功能：

| 功能 | 说明 |
|------|------|
| 节点接入 | 注册和管理边缘节点，查看节点状态，远程执行命令和文件管理 |
| 任务管理 | 创建批量任务，对多个节点同时执行命令、读写文件或运行 Playbook |

### 快速导航

| 文档 | 说明 |
|------|------|
| OTA 简介 | OTA 模块的功能介绍 |
| 节点接入 | 注册节点、查看节点详情、远程操作 |
| 任务管理 | 创建和管理批量运维任务 |
| 快速上手 | 注册第一个节点并创建第一个任务 |
| 常见问题 | 节点离线、任务失败等问题的排查 |

## 常见问题

### 平台基础

#### Q: 边缘智能管理平台支持哪些边缘计算框架？

A: 平台支持以下边缘计算框架：
- **KubeEdge**：云原生边缘计算框架
- **OpenYurt**：阿里开源的边缘计算平台
- **vCluster**：虚拟集群方案

#### Q: 平台支持哪些浏览器？

A: 推荐使用以下浏览器访问：
- Chrome 90+（推荐）
- Firefox 90+
- Safari 14+
- Edge 90+

#### Q: 如何重置管理员密码？

A: 请联系系统管理员通过后端工具重置密码。平台不支持自助找回密码功能。

### 集群管理

#### Q: 如何导入已有的 Kubernetes 集群？

A: 参见创建第一个集群文档中的集群导入章节。

#### Q: 边缘节点断网后还能正常工作吗？

A: 是的，边缘节点具备离线自治能力。断网期间：
- 已部署的工作负载继续运行
- 节点本地数据持续采集
- 网络恢复后自动同步状态

#### Q: 节点组和工作空间有什么区别？

A:
- **节点组**：按物理位置或功能对节点进行分组，是资源分配的基本单元
- **工作空间（租户空间）**：为不同团队或项目提供逻辑隔离环境，是权限管理的基本单元

### 应用管理

#### Q: 如何上架自定义应用到应用商店？

A: 参见应用商店使用指南中的应用上架流程。

#### Q: 应用部署失败如何排查？

A: 参见应用故障排查指南，常见原因包括：
- 资源不足（CPU/内存/存储）
- 镜像拉取失败
- 配置错误
- 网络问题

#### Q: 支持哪些类型的应用包？

A: 平台支持 **Helm Chart** 格式的应用包。

### 权限管理

#### Q: 平台有哪些内置角色？

A: 请参见权限管理详细指南中的权限矩阵章节，其中列出了所有内置角色及其权限。

#### Q: 如何为不同团队设置独立的权限？

A: 通过以下方式实现：
1. 为每个团队创建独立的工作空间（租户空间）
2. 在工作空间内设置成员角色
3. 配置资源配额限制

#### Q: 权限变更后为什么没有生效？

A: 权限变更通常会立即生效。如果未生效，请尝试：
1. 重新登录平台
2. 清除浏览器缓存
3. 检查角色绑定是否正确配置

### GPU 算力

#### Q: 如何启用 vGPU 功能？

A: 需要在集群中部署 vGPU 调度组件，并创建相应的算力配比模板。参见算力管理。

#### Q: 平台支持哪些 GPU 型号？

A: 平台支持所有主流 NVIDIA GPU。

### IoT 设备

#### Q: 如何接入新的物联网设备？

A: 参见设备管理中的设备注册章节。需要先创建设备模版，再注册具体设备。

#### Q: 支持哪些设备通信协议？

A: 目前支持 MQTT、Modbus、OPC UA 以及自定义协议扩展。

### OTA 升级

#### Q: OTA 升级失败如何处理？

A: 参见 OTA 故障排查。常见原因：
- 边缘节点网络不稳定
- 存储空间不足
- 升级包校验失败

### 镜像管理

#### Q: 如何配置私有镜像仓库？

A: 参见镜像仓库管理文档。

#### Q: 镜像拉取失败如何排查？

A: 参见镜像故障排查，常见原因：
- 认证信息不正确
- 网络不通
- 镜像地址错误
