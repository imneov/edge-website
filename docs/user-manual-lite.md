## 欢迎使用

边缘智能管理平台是一款面向云原生应用的智能边缘计算平台，支持 Kubernetes 分布式边缘基础设施的无缝管理。

### 核心功能

- **云原生边缘管理**：使用标准 Kubernetes API 管理边缘节点
- **分布式架构**：支持地理分布式边缘场景
- **智能编排**：自动化工作负载部署和生命周期管理
- **统一可观测性**：跨云端和边缘的集中监控和日志
- **安全与合规**：内置安全策略和合规控制

---

## 系统架构

平台采用云边协同架构：

- **云控制平面**：edge-apiserver、edge-controller、edge-console
- **监控服务**：Prometheus、Alertmanager、Grafana
- **边缘组件**：Kubernetes 集群、Edge Agent

---

## 前置要求

### 客户端要求

- 操作系统：Windows 10+, macOS 10.15+, Linux (Ubuntu 18.04+)
- 浏览器：Chrome 90+, Firefox 88+, Safari 14+
- 网络：稳定的互联网连接

### 服务器要求（最小配置）

- 节点数量：1 个控制平面 + 2 个工作节点
- CPU：每节点 4 核
- 内存：每节点 8GB
- 存储：每节点 100GB SSD

### 软件依赖

- Kubernetes ≥ 1.24.0（推荐 1.28.x）
- OpenYurt ≥ 1.4.0（推荐 1.6.x）
- 存储后端：NFS 或 Ceph RBD

## 创建第一个集群

### 步骤

1. 准备 kubeconfig 文件
2. 登录 Console，进入"集群管理"
3. 点击"添加集群"按钮
4. 填写集群信息并上传 kubeconfig
5. 点击"测试连接"验证
6. 确认添加

### 集群状态

- 🟢 正常：集群运行正常
- 🟡 告警：部分组件异常
- 🔴 离线：无法连接
- 🔄 同步中：正在同步状态

## 常见问题

### 登录问题

**Q: 忘记密码怎么办？**  
A: 点击"忘记密码"链接，或联系管理员重置。

**Q: 账号被锁定？**  
A: 连续输错密码 5 次会被锁定，等待 30 分钟或联系管理员。

### 集群���题

**Q: 集群显示离线？**  
A: 检查集群 API Server 是否运行，验证网络连通性。

**Q: 节点组和工作空间有什么区别？**  
A: 节点组按物理位置分组节点，工作空间为团队提供逻辑隔离。

### 应用问题

**Q: 应用部署失败？**  
A: 检查资源是否充足、镜像是否可拉取、配置是否正确。
